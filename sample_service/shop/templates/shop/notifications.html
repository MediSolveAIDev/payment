🔍 개발자 노트 — 서비스 알림 수신 POST /notify

수신 엔드포인트

POST /notify — 결제 서버 → 이 서비스. JSON 본문 + 헤더 X-Event(이벤트명) · X-Signature/X-Timestamp/X-Nonce(서명).

서명·등록

  • 서명 검증: HMAC_SHA256(hmac_secret, "POST\n{path}\n{ts}\n{nonce}\n{sha256(body)}") — API 호출 서명과 동일 방식. 불일치 시 거부(운영). (이 샘플은 데모라 불일치도 기록만 하고 200)
  • 등록: 결제 서버 어드민 → 서비스 상세 → 서비스 알림 URL에 https://이서비스/notify 입력(어드민 '테스트 알림'으로 즉시 확인)

전달 규약(중요)

  • 재시도 없음(at-most-once) — 단발 POST라 수신 실패(다운·타임아웃·5xx) 시 그 이벤트는 영구 유실. 중요 상태는 알림에만 의존하지 말고 조회 API(GET /api/v1/subscriptions·/payments/{external_user_id})로 보완
  • 응답: 2xx를 빠르게 반환(본문은 무시됨). 발송 타임아웃 5초 → 무거운 처리는 수신 후 비동기로
  • 멱등: 전용 전달ID 없음 → 결제는 order_id, 구독은 subscribe_id+EVENT+STATUS+date로 중복 판별(X-Nonce는 발송마다 랜덤이라 중복키로 쓰지 말 것)

payload 키 & 이벤트

  • 키: EVENT(=X-Event 헤더값) · STATUS/PRE_STATUS · DESC · service_name · date(KST) · subscribe_id(구독 ID — subscription_id 아님) · order_id · email(실제론 external_user_id, 이메일 아닐 수 있음)
  • 이벤트(17종+테스트): subscription.*(created/status_changed/renewed/force_canceled/extended) · card.*(registered/replaced/deleted/activated/deactivated) · payment.one_off* · plan.* · notification.test
