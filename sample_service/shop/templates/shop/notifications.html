🔍 개발자 노트 — 서비스 알림 수신 POST /notify

수신 엔드포인트

POST /notify — 결제 서버 → 이 서비스. JSON 본문 + X-Signature/X-Timestamp/X-Nonce 서명.

핵심 규칙

  • 서명 검증: HMAC_SHA256(hmac_secret, "POST\n{path}\n{ts}\n{nonce}\n{sha256(body)}") — API 호출 서명과 동일 방식
  • 등록: 결제 서버 어드민 → 서비스 상세 → 서비스 알림 URL에 https://이서비스/notify 입력
  • payload의 EVENT로 이벤트 종류를 구분하고, DESC에 금액 등 상세가 담긴다

구현 위치

  • shop/views.py → notify_receive_view() / _verify_notify_signature()
