🔍 개발자 노트 — 구독 상태 머신과 라이프사이클 API 5개 엔드포인트

호출하는 API (모두 payment_client.py 한 줄 호출)

GET /api/v1/subscriptions/{external_user_id} — 조회 get_subscription()
POST …/cancel — 취소(만료일까지 유지) cancel()
POST …/resume — 만료 전 재개 resume()
POST …/pay — 수동 결제(연체·정지 복구) manual_pay()
POST /api/v1/cards — 카드 변경(재등록=교체) register_card() · 별도 change-card 엔드포인트는 제거됨

상태 머신 — 배지 색과 동일

  • TRIAL 체험 → 만료 시 자동 첫 결제 → ACTIVE
  • ACTIVE 정상 → 갱신 실패 시 PAST_DUE(연체·접근 허용) → 재시도 소진 시 SUSPENDED(정지·접근 차단)
  • CANCELED 취소 예약(만료일까지 이용) → 만료 시 EXPIRED(종단) / 만료 전 재개 가능
  • 수동 결제 성공 시 PAST_DUE·SUSPENDED 모두 ACTIVE로 복귀(결제 기준일 리셋)

구현 위치

  • 화면 로직: shop/views.py → my_view(), 액션 공통 처리 _action_view()
  • 접근 제어는 응답의 access_allowed를 쓰면 된다 — 상태별 분기를 직접 구현할 필요 없음
