🔍 개발자 노트 — 이 화면의 API·코드 GET /api/v1/services

호출하는 API

GET /api/v1/services — 서비스 목록(id·이름·상태). 무인증 엔드포인트.

구현 위치

  • API 호출: shop/payment_client.py → list_services()
  • 화면 로직: shop/views.py → services_view() / 키 저장 service_save_key_view()
  • 키 보관: shop/models.py → ServiceCredential (서비스별 api_key·hmac_secret를 로컬 DB에 저장)

핵심 규칙

  • API 키·HMAC Secret은 서비스 등록 시 1회만 평문 발급 — 분실 시 어드민에서 재발급(rotate)
  • 키가 서버에서 재발급되면 다음 호출이 401 → 이 화면으로 자동 복귀해 재입력(?reauth=)
  • 실서비스에서는 키를 화면 입력이 아니라 환경변수/시크릿 매니저로 주입하세요
  • 새 흐름: 이메일 선택(/login) 후 이 화면 진입 → 서비스 선택 완료 시 카드 등록(/card)으로 이동
