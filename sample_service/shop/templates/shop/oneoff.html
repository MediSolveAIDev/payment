🔍 개발자 노트 — 단건 결제와 금액 변조 방지 POST /api/v1/payments

호출하는 API

POST /api/v1/payments — 단건 결제 create_one_off_payment()
POST /api/v1/payments/{order_id}/cancel — 취소(환불) cancel_one_off_payment()

핵심 규칙 — 그대로 따라 하세요

  • 금액은 HMAC 서명된 본문으로 전달 — 중간자가 amount를 변조하면 서명 검증에서 거부된다 (shop/views.py → oneoff_view())
  • order_id는 서비스가 생성(oo-<uuid>) — 서비스 내에서만 고유하면 되고, 같은 order_id 재요청은 서버가 멱등 처리(이중결제 방지)
  • 카드 보관함 전환: 토스 authKey 불필요 — 사전 등록된 카드(POST /api/v1/cards)의 빌링키를 서버가 자동 조회해 청구한다. 카드 미등록 시 서버 404 → 카드 등록 화면으로 유도

구현 위치

  • 폼·결제 실행: shop/views.py → oneoff_view() (POST 시 즉시 create_one_off_payment() 호출)
  • 로컬 기록: shop/models.py → OneOffRecord (결제 내역 화면에서 취소 테스트용)
