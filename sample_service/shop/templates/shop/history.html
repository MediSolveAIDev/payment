🔍 개발자 노트 — 내역 조회와 취소 수수료 GET /api/v1/payments/{uid}

호출하는 API

GET /api/v1/payments/{external_user_id} — 구독+단건 결제 전체(최신 50건) get_payments()
POST /api/v1/payments/{order_id}/cancel — 단건 취소 cancel_one_off_payment()

핵심 규칙

  • 응답의 각 결제에 cancelable / cancel_fee / cancel_refund_amount / cancel_fee_percent가 포함 — 취소 전엔 예상값, 취소 후엔 실제값. 수수료 계산을 직접 구현하지 말 것
  • 단건 결제의 상품명·취소여부는 이 서비스의 로컬 DB(OneOffRecord)와 order_id로 매칭해 표시
  • 취소 정책이 꺼진 서비스(cancellation_enabled=False)는 취소 시 CANCEL_DISABLED 에러가 반환된다

구현 위치

  • shop/views.py → history_view() (두 데이터 결합 로직 주석 참고)
