🔍 개발자 노트 — 카드 보관함 API 3종 POST / GET / DELETE /api/v1/cards

호출하는 API (payment_client.py)

POST /api/v1/cards — 카드 등록/교체 register_card() · 재등록이 곧 "카드 변경"
GET /api/v1/cards/{external_user_id} — 등록 카드 조회 get_card()
DEL /api/v1/cards/{external_user_id} — 카드 삭제 delete_card()

흐름 (등록 버튼을 누르면)

  • ① 프론트: 토스 SDK requestBillingAuth()로 카드 등록창 — 카드번호는 토스만 다룬다
  • ② 토스 → successUrl: authKey·customerKey를 /billing/success로 전달
  • ③ 백엔드: authKey로 POST /api/v1/cards 호출 → 빌링키 발급·보관(서버가 마스킹 카드만 반환)

핵심 규칙(서버 카드 보관함 전환)

  • 구독·단건결제는 더 이상 authKey가 필요 없다 — 먼저 카드를 등록해 두면 서버가 그 카드로 결제한다
  • 카드 변경 = 같은 사용자로 POST /api/v1/cards 재호출(기존 카드 교체). 별도 change-card 엔드포인트는 제거됨
  • 사용 중인 구독이 있는 카드는 삭제 시 409 CONFLICT

구현 위치

  • 화면/콜백: shop/views.py → card_view(), billing_success_view()
  • API 호출: shop/payment_client.py → register_card/get_card/delete_card
