# 요청 001 처리 계획

원본: docs/requests/001.md

## 확정된 결정
- 반복회차/구독시작일/만료일 = **표시 전용**(유한구독 모델 변경 없음). 구독/요금제 상세에 결제주기·지금까지 결제 회차·시작일·만료일 표시.
- 관리자↔서비스 **다대다** 도입(1명이 여러 서비스). 기존 User.service_id는 '주 서비스'로 유지 + user_services 연결테이블로 추가 부여. 유효 스코프 = 주 서비스 ∪ 추가부여.
- **단계별** 구현(각 단계 테스트+커밋+보고).

## 단계
1. **공통 페이징/검색/필터/정렬** — 전 목록 15개/페이지, 검색·상태필터·정렬. 재사용 헬퍼 + 매크로.
2. **계정 관리 + 다대다** — 계정 추가 UI, user_services 연결테이블·마이그레이션, 다중 서비스 스코프, 서비스에 관리자 할당.
3. **서비스 상세에서 요금제 관리** — 요금제 CRUD를 서비스 상세에 내장(수정/비활성/삭제), 할인율 입력, 반복회차·시작·만료 표시, 키 복사 버튼.
4. **감사로그 가독화** — 액션코드→한글 설명, 행위자 UUID→이메일, 대상 친화 표기.
