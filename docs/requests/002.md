서비스의 상태에 대한 요청이다.

1. 서비스 요금제
   1. 각요금제에 Trial
      1. 결제정보가 등록되어 있을 경우 신청가능
      2. 가부를 체크하고, Trial로 사용가능한 날짜를 세팅
      3. Trial 만료시 신청한 요금제 자동결제
      4. 사용자가 Cancel하면 Trial은 즉시 서비스 종료 Canceled

상태값 정의

| 상태 (Status)                                                                                     | 서비스 접근 권한              | 다음 결제 스케줄 및 기준일 정책                     | 진입 조건 및 상태 전이(Transition) 로직                                       |
| ------------------------------------------------------------------------------------------------- | ----------------------------- | --------------------------------------------------- | ----------------------------------------------------------------------------- |
| 1. Trial                                                                                          |                               |                                                     |                                                                               |
| (체험 기간)                                                                                       | O (가능)                      | 체험 만료 시점에 첫 정기 결제 시도                  | [진입] 결제 정보 등록을 동반한 체험 시작 시 진입                              |
| [전이] 지정된 체험 기간 만료 시 자동 결제 수행 -> 성공 시 Active, 실패 시 Past_Due                |                               |                                                     |                                                                               |
| 2. Active                                                                                         |                               |                                                     |                                                                               |
| (정상 이용)                                                                                       | O (가능)                      | 설정된 결제 기준일(Billing Anchor)에 다음 주기 결제 | [진입] 결제 성공 시 진입 (최초 결제, 재결제 등)                               |
| [전이] 정기 결제 실패 시 Past_Due, 사용자가 해지 예약 시 Canceled                                 |                               |                                                     |                                                                               |
| 3. Past_Due                                                                                       |                               |                                                     |                                                                               |
| (결제 실패/유예)                                                                                  | O (유예 제공)                 | 12시간 단위로 최대 4회 재시도                       |                                                                               |
| (간격 및 횟수는 .env에서 관리)                                                                    | [진입] 정기 결제 실패 시 진입 |                                                     |                                                                               |
| [전이] 재결제 성공 시 Active 복귀, 4회 모두 실패 시 Suspended 차단                                |                               |                                                     |                                                                               |
| ※ 유예 기간 동안의 서비스 사용료는 면제(Forgiveness) 처리                                         |                               |                                                     |                                                                               |
| 4. Suspended                                                                                      |                               |                                                     |                                                                               |
| (강제 정지)                                                                                       | X (차단)                      | 시스템 결제 시도 중지, 사용자의 수동 결제 대기      | [진입] Past_Due에서 재결제 최종 실패 시 진입                                  |
| [전이] 설정된 대기 일자(기본 30일, .env 관리) 초과 시 Expired.                                    |                               |                                                     |                                                                               |
| [복귀] 사용자가 결제 성공 시 Active 전이 및 결제 기준일(Billing Anchor)을 해당 날짜로 리셋        |                               |                                                     |                                                                               |
| 5. Canceled                                                                                       |                               |                                                     |                                                                               |
| (해지 예약)                                                                                       | O (유지)                      |                                                     |                                                                               |
| (이번 주기 만료일까지)                                                                            | 다음 주기 결제 시도 안 함     | [진입] 사용자가 명시적으로 구독 해지 요청 시 진입   |                                                                               |
| [전이] 현재 결제 주기의 만료일(expires_at)이 지나는 즉시 배치 스케줄러에 의해 Expired로 강제 전이 |                               |                                                     |                                                                               |
| 6. Expired                                                                                        |                               |                                                     |                                                                               |
| (완전 종료)                                                                                       | X (차단)                      | 결제 스케줄러에서 영구 제외                         | [진입] Suspended 대기 시간 초과 또는 Canceled 만료일 도달 시 진입 (종단 상태) |
| (데이터는 즉시 삭제하지 않고 별도의 Data Retention 정책에 따라 보관)                              |                               |                                                     |                                                                               |
