서비스의 구독 및 결제에 대해서 어떤 상태의 변화가 발생할 경우, 해당 서비스에 알림을 주기 위한 기능

1. 서비스에서 알림을 받을 API URL을 등록
   1. 서비상세내역에 서비스알림을 받을 API를 입력하는 박스하나 생성(위치는 일반결제 취소정책 옆)

2. 알림을 보내는 경우 - 아래의경우일때 자동전송 (아래 경우에 대해서 화면에 설명이 필요함 - 상세버튼을 누르면 모달창으로 나와서 설명이 되어야함)
   1. 새로운 구독자 발생
   2. 구독관련 상태값 변화
   3. 구독 자동결제 발생
   4. 사용자가 카드 등록
   5. 사용자 카드 변경
   6. 사용자 카드 취소
   7. 관리자가 카드를 활성화, 비활성화
   8. 사용자가 일반 결제 했을때
   9. 사용자가 일반결제 취소했을때
   10. 관리자가 일반결제 결제 취소할때

3. API호출시 구조 (없는 값은 비우면됨)
   {
   "subscribe_id":"", #구독 ID(KEY)
   "order_id":"", # 주문번호(구독 및 일반결제에 대한 결제 주문번호),
   "PRE_STATUS":상태값, #구독/결제에대한 이전상태 값
   "STATUS":상태값, #구독/결제에대한 새로운 상태 값
   "service_name":"서비스이름,
   "email":"관련사용자 이메일",
   "date":"발생 년월일시간초"
   "DESC":"상세설명" # 금액등 상세한 설명이 포함
   }
