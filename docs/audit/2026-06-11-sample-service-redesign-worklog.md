## 1. 디자인 — Centurion Suite 디자인 시스템 적용

`docs/design/centurion-suite-handoff` 기준:

- **토큰**: Pretendard(가변 폰트 CDN), gray-100~800, primary `#476CFF`(+100/300/500), red `#FF4E51`(+100/300). 결제 상태 표현용 보조색(green/amber)은 샘플 확장 토큰으로 주석 명시
- **컴포넌트** (`base.html` 한 파일 — 외부 CSS 의존 없음, 그대로 복사 가능):
  - 스티키 헤더(브랜드 마크 + 활성 메뉴 표시 + 현재 서비스 칩 + 사용자)
  - 버튼 4종(primary/ghost/danger/sub — Button_L/S 스펙), 인풋(50px, focus=primary)
  - **상태 배지** — 구독/결제 상태 머신과 1:1 색 매핑(`st-ACTIVE`, `st-PAST_DUE` 등)
  - 테이블·메시지·온보딩 스테퍼·카드 등장 애니메이션(순차 페이드업, reduced-motion 대응)
- 금액은 humanize `intcomma`로 천단위 콤마 표시(`settings.py`에 앱 추가)

## 2. 교보재 컨셉 — 「개발자 노트」 (화면이 곧 문서)

모든 화면 하단에 접이식 다크 패널 추가. 각 화면이:

| 항목 | 내용 |
|---|---|
| 호출하는 API | 메서드 칩(GET/POST) + 경로 + 대응하는 `payment_client.py` 함수명 |
| 구현 위치 | `shop/views.py` 뷰 함수, 관련 모델·템플릿 |
| 핵심 규칙 | 타임아웃(503)≠실패, 금액 세션 보관(변조 방지), 멱등 order_id, `access_allowed` 사용 등 — 구독서버 연동 시 반드시 지킬 규칙을 해당 화면에서 바로 노출 |

특히 **카드 등록 화면은 기본 펼침** — "버튼을 누르면 일어나는 3단계(SDK 인증 → successUrl → 서버 API)"를 결제 직전에 보여준다. 내 구독 화면은 상태 머신 전체(TRIAL→ACTIVE→PAST_DUE→SUSPENDED, CANCELED→EXPIRED/재개)를 배지 색과 함께 설명.

## 3. UI/UX 개선

- **온보딩 스테퍼**: 서비스 선택 → 로그인 → 요금제 → 카드 등록 4단계 진행 표시
- **상태별 안내 배너**(내 구독): PAST_DUE(연체—수동 결제 유도), SUSPENDED(정지—복구 방법), CANCELED(재개 가능 안내) — 상태마다 가능한 동작 버튼이 달라지는 것을 명시적으로 설명
- 요금제 카드: 첫 결제가(강조)/정기 결제가/정가(취소선) 구분, 체험·자동결제안함 배지
- 결제 확인·결과·실패 화면을 중앙 패널로 통일, 빈 상태(구독 없음 등) 일러스트형 처리
- 401 키 재입력(reauth) 흐름: 해당 서비스 카드 빨간 테두리 + 폼 자동 펼침(기존 동작 유지, 시각 강화)

## 4. README 개편

- **「화면이 곧 문서」** 섹션 — 개발자 노트 사용법
- **「내 서비스에 가져다 쓰는 법」** 표 — `payment_client.py` 단일 파일 복사가 연동의 전부임을 명시(Django 의존 없음), 화면 패턴/토스 SDK 호출부/디자인의 참조 위치
- **연동 핵심 규칙 4가지** — 개발자 노트와 동일 내용을 문서로도 제공
