# 인수인계 가이드(new_manual) 신규 작성

- **일자**: 2026-06-11
- **요청**: 초보 개발자가 유지보수·기능추가를 할 수 있는 인수인계 문서를 HTML로, `docs/new_manual`에 생성
- **성격**: 기존 dev_manual(기능별 상세 레퍼런스)과 역할 분담 — new_manual은 **읽는 순서가 있는 학습 경로형 가이드**

## 구성 (마크다운 12장 + 빌드 스크립트 → index.html 단일 페이지)

| 그룹 | 장 |
|---|---|
| 시작 | README(사용법·3대 원칙·문서 관계) |
| 첫 주 — 순서대로 | 01 첫날 셋업(체크리스트), 02 큰그림(4 진입점·요청의 일생·레이어 규칙), 03 도메인(상태머신 7종·결제 3원칙·돈 계산 규칙), 04 코드 지도(변경 유형별 진입점·모듈 책임·코딩 규약) |
| 작업 | 08 기능 추가 레시피 4종(컬럼 추가/외부 API/어드민 화면/**구독 상태 추가 — EXTENDED 실사례 체크리스트 11항목**) |
| 사전 | 05 DB·마이그레이션, 06 인증·보안·배포 체크리스트, 07 배치·장애 대응(증상별), 09 테스트(FakeToss·동시실행 금지), 10 FAQ |
| 주의 | 11 알려진 이슈(로직 검증 High 4·정책 결정 대기·보류 항목·EXTENDED WIP 잔여) — audit 리포트 연결 |

## 특징

- dev_manual과 동일한 빌드 파이프라인(마크다운 → build_html.py → 단일 HTML, 사이드바·검색·고대비 CSS 재사용) — 유지보수 방식 통일
- 내용은 이 세션의 감사(Phase 1~4)·로직 검증에서 확립된 사실 기반 — 최근 변경(transitions 중앙화, toss_order_id, 배치 병렬화, EXTENDED WIP)까지 반영
- 11장이 "고치기 전에 반드시" 읽는 장으로 미해결 이슈·정책 결정 대기를 인수 항목으로 명시
- dev_manual README에 역방향 안내 링크 추가(+manual.html 재빌드)

## 검증

- 빌드 산출물 브라우저 확인: 사이드바 그룹 5종·섹션 12개·내부 .md 링크 앵커 재작성·외부 링크 보존, 표/상태머신 다이어그램/콜아웃 렌더링 정상
- 문서 내 실행 명령(create-admin·seed_demo·빌드 명령)은 코드/README와 대조 검증
