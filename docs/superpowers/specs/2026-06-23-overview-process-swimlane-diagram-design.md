# 개요 0-2 전체 프로세스 역할 스윔레인 그림 — 설계

- 날짜: 2026-06-23
- 상태: 승인됨 → 구현

## 목적

user_manual `00-overview.md` 0-2절의 전체 프로세스(1~9단계)를 **역할 스윔레인 다이어그램**(인라인 SVG)으로도 표현해, "어느 역할이 어느 단계를 하는지"를 한눈에 보이게 한다. 기존 단계 카드(`.flow`)·개발자 연동 시퀀스(`.seqwrap`)는 그대로 유지(보완).

## 결정

- 형태: **역할 스윔레인(가로 흐름)**. 4개 역할 레인 + 2개 구간 밴드(준비 1~5 / 일상 6~9), 1→…→9 단일 파란 경로.
- 스타일: 기존 `.seqwrap` 컨테이너/클래스(`lane`,`lane-t`,`life`,`msg`,`lbl`,`band`,`band-t`) 재사용 + 노드(번호 원·라벨)는 인라인 속성. **manual.css 수정 없음.**
- 배치: 0-2절 "함께 보기" 블록 뒤, "(개발자용) 연동 시퀀스" 앞.
- 접근성: `role="img"` + `aria-label`. 모바일 가로 스크롤(.seqwrap).
- 마커 id는 기존 SVG(`apr`/`agr`)와 겹치지 않게 별도 id(`fp`) 사용(같은 페이지 중복 방지).

## 레인·노드 매핑

| 레인(역할) | 단계 |
|---|---|
| 시스템 관리자 | ① 서비스 등록 · ② API 키 발급 · ③ 토스 시크릿 키 등록 |
| 서비스 담당자 | ④ 담당자 로그인 · ⑤ 요금제 생성 |
| 외부 서비스(앱) | ⑥ 카드 등록 · ⑦ 구독 요청 |
| 결제 서버·스케줄러 | ⑧ 자동 연장 · ⑨ 정산·대시보드·알림 |

구간 밴드: 준비(①~⑤) ‖ 일상(⑥~⑨).

## 구현/검증

- `00-overview.md`에 SVG 삽입 → `uv run --with markdown python docs/user_manual/build.py` 재빌드 → docker dev 이미지 재빌드(`docker compose -f docker-compose.dev.yml up -d --build`)로 `/user-manual/00-overview.html` 라이브 반영 → curl로 확인.
- 워크로그 docs/audit/.

## 비범위

- 기존 카드/시퀀스 SVG 변경·중복 제거 없음. manual.css 변경 없음.
