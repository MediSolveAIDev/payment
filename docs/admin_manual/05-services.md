# 05. 서비스 관리 (Services)

사내의 상점이나 앱을 결제 시스템에 등록하고, 각 서비스별 보안 및 운영 정책을 관리하는 메뉴입니다.

---

## 1. 서비스 등록 및 개요

새로운 서비스를 등록하여 통합 결제 인프라를 사용할 수 있게 합니다.

### 테이블 항목(컬럼) 상세 설명
*   **서비스명**: 등록된 서비스 이름입니다. 클릭 시 상세 관리 화면으로 이동합니다.
*   **요금제**: 해당 서비스에 등록된 전체 요금제 개수입니다.
*   **구독**: 현재 해당 서비스를 이용 중인 총 구독 건수입니다.
*   **상태**: 서비스의 활성/비활성 상태입니다.

---

## 2. 보안 및 연동 키 관리

서비스 연동에 필수적인 보안 데이터입니다. **시스템 관리자만 접근 및 관리가 가능합니다.**

*   **서비스 API Key**: 외부 서비스가 우리 시스템을 호출할 때 본인을 증명하는 아이디입니다. (`svc_` 시작)
*   **HMAC Secret**: 데이터 위변조 방지를 위한 비밀번호입니다. **생성 시 한 번만 보여지므로** 즉시 복사하여 안전한 곳에 보관해야 합니다.
*   **키 재발급 (Rotate Keys)**: Secret 키가 유출된 것으로 의심될 때 사용합니다. 재발급 즉시 기존 키는 무효화되므로, 연동된 서비스의 설정도 동시에 업데이트해야 합니다.

---

## 3. 담당자 및 알림 관리

*   **담당자 계정**: 해당 서비스의 데이터만 볼 수 있는 권한을 가질 관리자들을 지정합니다.
*   **대표 담당자 (Primary)**: 결제 실패나 갱신 오류 등 **시스템 알림 메일이 발송되는 주 수신처**입니다. 반드시 실시간 확인이 가능한 담당자를 대표로 지정하세요.

---

## 4. 보안 정책 (허용 IP)

결제 요청을 보낼 수 있는 서버의 IP 주소를 지정합니다.
*   허용되지 않은 IP에서 오는 결제 요청은 시스템이 즉시 거절합니다. 
*   운영 서버와 개발 서버의 IP를 각각 등록하여 보안 사고를 예방할 수 있습니다.

---

## 5. 취소 및 환불 정책 (Cancellation Policy)

해당 서비스에서 발생하는 일반 결제에 대한 환불 규칙을 설정합니다.
*   **취소 허용 여부**: 체크를 해제하면 고객이나 관리자가 시스템을 통해 환불할 수 없게 됩니다.
*   **취소 수수료 (%)**: 환불 시 공제할 수수료 비율입니다. (예: 10% 입력 시 1,000원 결제 취소 시 900원만 환불됨)

---

## 6. 이벤트 (감사 로그)

해당 서비스와 관련된 모든 설정 변경(요금제 수정, 담당자 변경 등) 이력이 하단에 실시간으로 기록됩니다. 누가 언제 어떤 항목을 변경했는지 투명하게 확인할 수 있습니다.
