# 03. 결제 관리 (Payments)

정기 구독 갱신과 일반 1회성 구매를 포함한 시스템의 모든 금전적 거래를 관리하고 환불을 처리하는 메뉴입니다.

---

## 1. 결제 내역 목록

개별 결제 건들의 성공 여부와 상세 정보를 확인할 수 있습니다.

### 테이블 항목(컬럼) 상세 설명
*   **승인 시각**: 결제가 최종 완료된 시각입니다.
*   **사용자**: 결제를 시도한 사용자의 ID입니다.
*   **주문번호**: 시스템이 생성한 고유 주문 번호입니다.
*   **유형**: `정기(Renewal)` 또는 `단건(One-off)` 구분입니다.
*   **종류**: `구독` 관련 결제인지 `일반` 결제인지 표시합니다.
*   **상태**: 결제 결과(승인/취소/실패)입니다.
*   **총매출**: 결제된 전체 금액입니다.
*   **환불**: 환불 처리된 경우 해당 금액이 빨간색으로 표시됩니다.
*   **순매출**: 총매출에서 환불액을 뺀 최종 금액입니다.

### 주요 항목 및 필드 설명
*   **주문번호 (Order ID)**: PG사(토스)와 통신 시 사용하는 고유 번호입니다. 결제 문의 시 가장 중요한 기준이 됩니다.
*   **유형**: 
    *   `정기(Renewal)`: 구독 요금제에 의해 매달/매년 반복적으로 발생하는 결제.
    *   `단건(One-off)`: 구독과 무관하게 1회성으로 발생하는 상품 구매 결제.
*   **금액**: 부가세가 포함된 총 결제 금액입니다.
*   **상태**: 
    *   `완료(DONE)`: 결제가 정상 승인되었습니다.
    *   `실패(FAILED)`: 결제 중 오류가 발생했습니다.
    *   `취소(CANCELED)`: 관리자에 의해 환불(취소) 처리되었습니다.

---

## 2. 결제 상세 정보

특정 결제 내역을 클릭하면 더 자세한 데이터를 볼 수 있습니다.

*   **실패 코드 & 메시지**: 결제가 실패한 경우, 카드사에서 보내온 정확한 이유(예: 잔액부족, 카드정지 등)를 확인할 수 있습니다.
*   **토스 결제키 (Payment Key)**: 토스페이먼츠 관리자 페이지에서 해당 거래를 직접 찾을 때 필요한 키 값입니다.
*   **승인 시각**: 카드사로부터 실제 승인이 완료된 초 단위 시각입니다.
*   **원본 응답 (Raw Response)**: PG사에서 보내온 데이터 전체를 보여줍니다. 일반 운영 시에는 무시해도 좋으나, 기술적 장애 분석 시 중요합니다.

---

## 3. 환불(취소) 처리 가이드

이미 완료된 결제에 대해 환불을 진행하는 절차입니다.

### 일반 단건 결제 취소
*   **취소 정책 안내**: 결제 상세 화면에 해당 서비스의 취소 정책(수수료율 등)에 따른 예상 환불 금액이 미리 표시됩니다.
*   **결제 취소 버튼**: 버튼을 클릭하면 실시간으로 PG사에 취소 요청이 전달됩니다.
*   **주의사항**: 이미 취소된 건은 다시 되돌릴 수 없습니다.

### 취소 완료 후 정보
*   취소가 완료되면 해당 결제의 상태가 `취소(CANCELED)`로 변경됩니다.
*   **환불액**: 고객에게 실제로 돌아간 금액입니다.
*   **취소 수수료**: 정책에 따라 서비스 운영자가 보유하게 된 수수료 금액입니다.

---

## 4. 증빙 관리 (영수증)

상세 화면에서는 고객에게 전달할 수 있는 **매출전표(영수증) 링크**를 제공합니다. 고객이 영수증 발행을 요청할 경우 이 링크를 복사하여 전달하면 카드사 공식 영수증을 확인할 수 있습니다.
