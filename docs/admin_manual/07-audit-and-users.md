# 07. 감사 및 계정 관리 (Audit & Users)

시스템의 안전한 운영을 위해 관리자 권한을 부여하고, 모든 활동 내역을 감시하는 시스템 관리자용 메뉴입니다.

---

## 1. 감사 로그 (Audit Log)

시스템 내부에서 발생하는 모든 주요 이벤트를 투명하게 기록합니다. 사고 예방과 사후 원인 파악을 위한 가장 강력한 도구입니다.

### 로그 검색 및 필터
*   **행위자**: 관리자가 직접 수행한 일인지, 외부 서비스 API가 요청한 일인지, 시스템 자동 스케줄러가 한 일인지 구분하여 검색할 수 있습니다.
*   **활동 유형**: 로그인, 요금제 변경, 환불 처리, 설정 변경 등 특정 행동 단위로 모아볼 수 있습니다.

### 상세 기록 항목(컬럼) 상세 설명
*   **시각**: 초 단위의 활동 발생 시각입니다.
*   **행위자**: 해당 작업을 수행한 주체입니다. (관리자 이메일, 외부 서비스명 등)
*   **활동**: 수행된 액션의 종류입니다. (예: 로그인, 요금제 수정 등)
*   **대상**: 변경이 가해진 리소스의 식별값입니다.
*   **상세 (Diff)**: 데이터의 구체적인 변경 내용입니다. 이전 값과 바뀐 값을 한눈에 비교할 수 있습니다.
*   **IP**: 요청이 발생한 접속자의 IP 주소입니다.

### 로그 보관 및 삭제
*   **엑셀 다운로드**: 모든 로그를 외부 파일로 보관할 수 있습니다.
*   **이전 로그 삭제**: 로그 데이터가 너무 많아질 경우, 특정 날짜 이전의 데이터를 일괄 삭제하여 시스템 최적화를 수행할 수 있습니다. (삭제 전 엑셀 백업 권장)

## 2. 계정 관리 (Users)

어드민 콘솔에 접속할 수 있는 운영자 계정을 관리합니다.

### 테이블 항목(컬럼) 상세 설명
*   **이메일**: 계정 아이디로 사용하는 이메일 주소입니다. 클릭 시 상세 정보 및 수정 화면으로 이동합니다.
*   **역할**: 부여된 권한 등급입니다. (시스템 관리자 / 서비스 담당자)
*   **주 서비스**: 해당 관리자가 전담하여 관리하는 서비스명입니다.
*   **상태**: 현재 계정의 활성화 상태입니다. (ACTIVE / PENDING / LOCKED)

### 권한(Role) 정의
*   **시스템 관리자 (SYSTEM_ADMIN)**: 모든 서비스의 데이터를 보고, 시스템 설정을 변경하며, 다른 계정을 생성할 수 있는 최고 권한입니다.
*   **서비스 담당자 (SERVICE_MANAGER)**: 본인에게 할당된 특정 서비스의 데이터만 조회 및 관리할 수 있습니다. 타 서비스의 결제나 정산 내역은 볼 수 없습니다.

### 계정 상태
*   `ACTIVE`: 정상 이용 가능.
*   `PENDING`: 계정 생성 후 아직 비밀번호 설정을 완료하지 않은 상태.
*   `LOCKED`: 비밀번호 5회 이상 실패로 인해 자동 잠금된 상태. (시스템 관리자가 해제해주어야 함)

---

## 3. 전체 설정 (System Settings)

시스템 전체에 영향을 미치는 글로벌 정책을 설정합니다.

*   **자동결제 재시도 설정**: 갱신 실패 시 몇 번이나 재시도할지(기본 3회), 몇 시간 간격으로 시도할지 설정합니다.
*   **어드민 접속 허용 IP**: 특정 사무실 IP에서만 어드민 로그인이 가능하도록 제한할 수 있습니다. (설정 시 본인의 IP를 반드시 포함하세요.)
*   **결제서버 킬스위치**: 긴급 점검 시 서비스 결제 API만 일시적으로 중단시킬 수 있습니다. 이때 어드민 기능은 계속 유지됩니다.
