# 02. 구독 관리 (Subscriptions)

사용자의 구독 상태를 추적하고, 결제 실패 대응이나 만료일 조정 등 운영상 필요한 직접적인 개입을 수행하는 핵심 메뉴입니다.

---

## 1. 구독 목록 및 검색

시스템에 등록된 모든 구독 데이터가 최신순으로 표시됩니다.

### 테이블 항목(컬럼) 상세 설명
*   **서비스**: 해당 구독이 속한 서비스(상점)의 이름입니다.
*   **사용자**: 외부 서비스에서 관리하는 사용자의 고유 ID입니다. 클릭 시 상세 화면으로 이동합니다.
*   **요금제**: 사용자가 구독 중인 상품의 이름입니다.
*   **상태**: 현재 구독의 진행 상태입니다. (배지 색상으로 구분)
*   **만료일**: 현재 결제 주기가 끝나는 날짜입니다.
*   **다음 결제**: 시스템이 자동으로 다음 회차 결제를 시도할 예정 시각입니다.

### 필터 및 검색 활용법
*   **사용자 검색**: 사용자 ID(External User ID)의 일부를 입력하여 특정 고객을 빠르게 찾을 수 있습니다.
*   **서비스/요금제 필터**: 특정 상품을 이용 중인 고객들만 모아서 볼 수 있습니다.
*   **상태 필터 (가장 중요)**:
    *   `체험(TRIAL)`: 무료 체험 중이며 아직 첫 결제가 발생하지 않은 고객.
    *   `활성(ACTIVE)`: 결제가 정상적으로 완료되어 서비스를 정상 이용 중인 고객.
    *   `미수(PAST_DUE)`: 결제일에 결제가 실패하여 시스템이 자동 재시도를 하고 있는 상태. (주의 깊은 확인 필요)
    *   `정지(SUSPENDED)`: 반복된 결제 실패로 인해 서비스 이용이 차단된 상태.
    *   `취소(CANCELED)`: 관리자가 강제 취소했거나 사용자가 해지하여 종료된 상태.
    *   `연장처리(EXTENDED)`: 관리자가 마케팅 등의 사유로 만료일을 임의로 늦춰준 상태.

---

## 2. 구독 상세 화면 (View Details)

목록에서 특정 항목을 클릭하면 나타나는 상세 화면에서는 고객의 모든 구독 정보를 확인하고 관리할 수 있습니다.

### 상세 정보 필드
*   **구독 ID (UUID)**: 시스템 내부 고유 번호입니다. 개발팀에 기술 문의를 할 때 이 값을 알려주면 빠른 확인이 가능합니다.
*   **결제 주기**: 요금제에 설정된 반복 주기와 현재까지 몇 번이나 결제가 성공했는지(반복회차) 보여줍니다.
*   **만료일 / 다음 결제 시각**: 현재 혜택이 언제 끝나는지, 다음 결제는 언제 시도되는지 보여줍니다.
*   **재시도 횟수**: 현재 주기에서 결제가 몇 번이나 실패하고 다시 시도되었는지 보여줍니다.
*   **카드 정보**: 결제에 사용 중인 카드의 번호 일부를 확인할 수 있습니다. (보안상 전체 번호는 노출되지 않습니다.)

---

## 3. 관리자 액션 (운영 기능)

상세 화면 우측의 '관리' 카드에서는 다음과 같은 강력한 기능을 제공합니다.

*   **즉시 재결제 (Retry Payment)**: 구독이 `미수` 또는 `정지` 상태일 때 사용합니다. 고객이 카드를 한도 증액 등으로 정리했다고 연락이 오면, 다음 자동 시도까지 기다리지 않고 **지금 즉시 결제를 시도**할 수 있습니다. 성공하면 구독은 즉시 `활성`으로 복구됩니다.
*   **만료일 연장 (Extend Expiry)**: 특정 날짜를 선택하여 만료일을 강제로 늦출 수 있습니다. 고객 보상이나 이벤트 당첨자 등에게 유용하며, 연장된 기간 동안은 자동 결제가 발생하지 않습니다.
*   **강제 취소 (Force Cancel)**: 관리자가 구독을 즉시 종료시킵니다. 이 버튼을 눌러도 이미 결제된 금액이 자동으로 환불되지는 않으므로, 환불이 필요하다면 '결제' 메뉴에서 별도로 진행해야 합니다.
*   **연장 이력 조회**: 과거에 관리자가 만료일을 언제, 누구에 의해, 어느 날짜로 변경했는지 기록을 확인할 수 있습니다.

---

## 4. 결제 이력 테이블 (하단)

해당 구독에서 발생한 모든 결제 시도 기록입니다.

*   **주문번호**: 각 결제 시도에 부여된 고유 번호입니다. (앞부분만 표시)
*   **유형**: `정기(Renewal)` 결제인지, `단건(One-off)` 결제인지 표시합니다.
*   **금액**: 해당 회차에 청구된 금액입니다.
*   **상태**: 결제 성공 여부입니다. (`DONE`: 성공, `FAILED`: 실패, `CANCELED`: 취소)
*   **실패 사유**: 결제 실패 시 카드사에서 보내온 오류 코드입니다. 마우스를 올리면 상세 내용이 표시됩니다.
*   **요청 시각**: 결제가 요청된 정확한 일시입니다.
