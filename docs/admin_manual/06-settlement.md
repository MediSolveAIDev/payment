# 06. 정산 관리 (Settlement)

기간별 매출을 집계하고, 서비스별로 실제로 정산받아야 할 금액을 확정하는 메뉴입니다.

---

## 1. 정산 요약 및 월 선택

화면 상단에서 조회하고자 하는 월 또는 특정 날짜 범위를 선택합니다.

*   **순매출 (Net Total)**: `총 결제액 - 환불액`으로 계산된 실제 정산 기준 금액입니다. (가장 중요)
*   **분해 통계**: 총매출, 환불액, 구독 매출, 일반 매출의 비중을 상세히 나누어 보여줍니다.

---

## 2. 전체 서비스 정산 목록

시스템 관리자가 전체 매출을 파악할 때 사용합니다.

### 테이블 항목(컬럼) 상세 설명
*   **서비스**: 각 상점의 이름입니다.
*   **건수**: 기간 내 발생한 전체 승인 및 취소 건수입니다.
*   **구독**: 정기 결제로 발생한 매출액입니다.
*   **일반**: 1회성 결제로 발생한 매출액입니다.
*   **총매출**: 취소 전 전체 승인 금액의 합계입니다.
*   **환불**: 기간 내 처리된 환불액의 총합입니다.
*   **순매출**: 총매출에서 환불을 뺀 실제 정산 대상 금액입니다.

---

## 3. 서비스별 상세 정산 내역

특정 서비스의 정산 상세 화면에서는 해당 기간에 발생한 모든 결제 건이 표시됩니다.

### 테이블 항목(컬럼) 상세 설명
*   **승인 시각**: PG사 승인이 완료된 시각입니다.
*   **사용자**: 해당 결제를 수행한 사용자 ID입니다.
*   **주문번호**: 고유 주문 식별 번호입니다.
*   **유형**: `Renewal`(정기) / `One-off`(단건) 표시.
*   **종류**: `구독` / `일반` 구분.
*   **상태**: `승인` / `취소` 상태 표시.
*   **총매출**: 해당 건의 결제 금액입니다.
*   **환불**: 취소된 경우 환불된 금액입니다.
*   **순매출**: 해당 결제 건으로부터 확보된 최종 금액입니다.

---

## 4. 엑셀(Excel) 데이터 추출

모든 정산 화면에는 **엑셀 다운로드** 버튼이 있습니다.
*   재무 부서와의 교차 검증이나 세무 처리를 위해 필요한 모든 데이터(주문번호, 사용자 ID, 승인번호, 카드사 등)가 엑셀 파일에 포함됩니다.

---

## 5. 집계 기준 유의사항

*   **승인일 기준**: 정산은 결제가 실제로 승인된 시점을 기준으로 합니다.
*   **KST vs UTC**: 시스템 내부적으로는 세계 표준시(UTC)를 사용하지만, 화면과 엑셀 리포트는 운영자의 편의를 위해 **한국 시간(KST)**으로 변환하여 보여줍니다. 다만, 월 경계(매월 1일 오전) 결제 시 전월/익월 귀속 여부를 꼼꼼히 확인하세요.
