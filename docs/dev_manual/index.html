기능별 상세

요청 → 파일/코드 → DB → 응답을 기능 단위로 추적.

0404. 구독 생성외부 서비스가 사전 등록된 카드(카드 보관함, POST /api/v1/cards)를 기반으로 구독을 생성한다. 0505. 구독 갱신·만료·재시도·정합성(스케줄러 배치)스케줄러가 5분마다 만료·취소·정지된 구독을 자동으로 결제하거나 종료하는 배치다. 0606. 구독 취소·재개·수동결제·카드변경이미 존재하는 구독을 취소 예약·재개·수동결제·카드 교체하는 4가지 라이프사이클 동작을 다룬다. 0707. 단건(일반) 결제 + 취소구독과 완전히 무관한 1회성 결제를 생성하고, 필요하면 취소(환불)하는 기능입니다. 0808. 요금제 관리 (생성·수정·금액 계산·자동결제안함·추가정보)관리자(SYSTEMADMIN 또는 SERVICEMANAGER)가 어드민 화면에서 요금제를 생성·수정·보관·활성화·삭제하며, 금액 계산(첫 결제 / 정기 결제 / 상시 할인)은 서버의 billingm 0909. 서비스 등록·키 발급/회전·취소정책·담당자사내 서비스를 결제/구독 API 서버에 등록하고, 외부 서비스가 API를 호출할 때 사용하는 1010. 정산(어드민 화면)기간·서비스·요금제로 필터링해 승인(DONE)+취소(CANCELED) 결제(approvedat 기준)를 서비스별로 합산하고, 총매출/환불/순매출과 구독·일반 매출을 분리 표시하는 조회 전용 어드민 1111. 대시보드 — 어드민 메인 화면어드민 콘솔에 로그인하면 가장 먼저 보이는 메인 화면입니다. 1212. 웹훅 처리 (토스페이먼츠 → 서버)토스페이먼츠가 결제 상태 변경·빌링키 삭제 등 이벤트 발생 시 서버로 HTTP POST를 밀어 보내는(push) 것이 웹훅이다. 1313. 어드민 계정·역할·로그인/비밀번호SYSTEMADMIN이 어드민 콘솔에서 관리자 계정(SYSTEMADMIN 또는 SERVICEMANAGER)을 1414. 전체 설정(재시도·어드민 IP·킬스위치)SYSTEMADMIN 전용 전체 설정 화면에서 자동결제 재시도 정책, 어드민 접속 허용 IP, 결제서버 킬스위치 세 가지를 DB 단일 행(globalsettings.id=1)에 저장하고, 각 설정이 1515. 외부 API 레퍼런스 + 샘플 서비스 연동사내 다른 서비스(예: 진료 앱, 쇼핑몰)를 구독결제 서버에 실제로 연동할 때 필요한 1616. 카드 보관함(Card Vault) — 카드 등록·교체·조회·삭제사용자의 결제수단(빌링키)을 서버에 안전하게 보관하고, 구독·단건 결제 시 재사용할 수 있도록 합니다. 1717. 서비스 알림(아웃고잉 웹훅) — 요청 016구독·결제·카드·요금제 상태가 바뀌면 서비스가 등록한 URL로 JSON 알림을 POST한다.
