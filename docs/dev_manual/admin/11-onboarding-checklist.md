# 신규 서비스 온보딩 체크리스트 (운영자용)

> 새로운 사내 서비스를 구독/결제 시스템에 연결할 때 **순서대로 따라 하는** 체크리스트입니다.
> 각 단계의 자세한 화면 설명은 대괄호 안 문서를 참고하세요.

## ⚠️ 시작 전 꼭 알아둘 것
- **담당자 계정을 먼저 만들어야 합니다.** 서비스 등록 화면은 담당자를 *선택*하는 방식이라, 계정이 없으면 서비스에 담당자를 지정할 수 없습니다. → **계정 생성(1단계)이 서비스 등록(2단계)보다 먼저.**
- **API 키·HMAC 시크릿은 발급 직후 화면에서 단 한 번만 표시됩니다.** 그 자리에서 복사해 안전하게 보관·전달하세요. 분실하면 재발급(키 회전)해야 하고, 그러면 외부 서비스가 키를 다시 설정해야 합니다.
- **허용 IP는 1개 이상 필수**입니다(IPv4). 비워둘 수 없으며, 잘못 넣으면 외부 서비스의 API 호출이 전부 차단됩니다.

---

## 0단계 — 정보 수집 (등록 전 준비)
아래 정보를 서비스 담당 부서로부터 미리 받아 둡니다.

- [ ] **서비스명** (콘솔에 표시될 이름, 중복 불가)
- [ ] **구독/결제 담당자 이메일** (로그인 계정이 됨, 1명 이상 — 대표 담당자 1명 지정)
- [ ] **API를 호출할 서버의 공인 IP** (IPv4, 1개 이상 — 허용 IP 화이트리스트)
- [ ] **일반(단건) 결제 취소 정책** — 취소 허용 여부 / 취소 수수료율(0~100%)
- [ ] **요금제 안** — 요금제명, 가격, 결제주기(년/월/주/일), 첫구독 할인·무료, 상시 할인, 체험 제공 여부, 자동결제 여부

---

## 1단계 — 담당자 계정 생성  [→ 08-users.md]
서비스 담당자가 콘솔에 로그인하고 자기 서비스를 관리할 수 있도록 계정을 먼저 만듭니다.

- [ ] **계정 → 계정 추가**에서 담당자 이메일 입력, 역할 = **SERVICE_MANAGER** 선택
- [ ] 저장하면 담당자에게 **비밀번호 설정 링크(토큰)** 가 이메일로 발송됨 — 담당자가 직접 비밀번호를 설정
- [ ] 담당자가 여러 명이면 인원수만큼 반복 생성
- [ ] (확인) 담당자가 비밀번호 설정 링크로 로그인 가능한지 안내

> 비밀번호 설정 링크는 **48시간 후 만료**·**1회용**입니다. 만료되면 계정 상세에서 **비밀번호 재설정**으로 새 링크를 보낼 수 있습니다.

---

## 2단계 — 서비스 등록  [→ 03-services.md]
- [ ] **서비스 → 서비스 등록**(`/admin/services/new`)에서 입력:
  - [ ] 서비스명
  - [ ] **허용 IP** (옥텟 4칸·숫자만, 행 추가로 여러 개 — 1개 이상 필수)
  - [ ] **담당자 선택** + **대표 담당자 지정**(대표 = 알림 수신처)
  - [ ] **취소 정책**: 취소 허용 체크 / 취소 수수료율(%)
- [ ] 등록을 누르면 **키 발급 화면**으로 이동

---

## 3단계 — 키 발급·전달 (1회성, 중요)  [→ 03-services.md]
- [ ] 키 발급 화면에서 **API 키**와 **HMAC 시크릿**을 각각 **복사**
- [ ] 두 값을 **안전한 채널**(사내 비밀 저장소 등)로 서비스 담당자에게 전달
  - 메신저/이메일 평문 전송은 지양 — 노출되면 결제 요청에 악용될 수 있음
- [ ] 전달 완료 후 화면을 벗어남 (이후에는 평문 키를 다시 볼 수 없음)

> 키를 분실/유출했다면: 서비스 상세 → **키 재발급(회전)**. 새 키가 즉시 발급되고 **이전 키는 무효화**되므로, 외부 서비스가 새 키로 다시 설정해야 합니다.

---

## 4단계 — 요금제 생성  [→ 04-plans.md]
구독을 받으려면 **활성(ACTIVE) 요금제**가 최소 1개 있어야 합니다. (단건 결제만 쓸 거면 생략 가능)

- [ ] 서비스 상세의 **요금제 탭** 또는 **요금제 → 추가**에서 생성:
  - [ ] 이름·가격·**결제주기**(일 선택 시 일수 입력)
  - [ ] (선택) **첫구독 혜택**: 무료 / 할인 금액 / 할인율
  - [ ] (선택) **상시 할인**
  - [ ] (선택) **체험 제공**(체험 일수) / **자동결제 여부**
  - [ ] 금액 미리보기로 첫 결제·정기 결제 금액 확인 후 저장
- [ ] 요금제 상태가 **ACTIVE**인지 확인 (비활성 요금제로는 구독 불가)

---

## 5단계 — 외부 서비스 연동  [→ ../15-external-api-and-sample.md]
- [ ] 서비스 담당(개발자)에게 **API 베이스 URL + API 키 + HMAC 시크릿** 전달
- [ ] **요청 서명(HMAC) 방식** 안내: 헤더 4종(`x-service-key`/`x-timestamp`/`x-nonce`/`x-signature`)과 서명 정준 문자열 (상세·예시는 15번 문서, 인증 원리는 [../03-auth-and-security.md])
- [ ] 호출 서버의 공인 IP가 **허용 IP에 포함**되어 있는지 재확인(누락 시 401/403)
- [ ] (권장) **샘플 서비스**(`sample_service/`)로 연동을 먼저 시연·테스트 — 서비스 선택 → 키 입력 → 구독/결제 테스트

---

## 6단계 — 동작 검증
실제 운영 전에 테스트로 한 번 흐름을 확인합니다.

- [ ] 외부 서비스(또는 샘플)에서 **테스트 구독 1건** 생성 → 콘솔 **구독** 화면에 ACTIVE(또는 TRIAL)로 보이는지 [→ 05-subscriptions.md]
- [ ] (단건 결제를 쓴다면) **테스트 단건 결제 1건** → **결제** 화면에 DONE으로 보이는지 [→ 06-payments.md]
- [ ] **결제** 화면에서 실패 건이 있으면 **실패 코드에 마우스를 올려** 원인 확인
- [ ] **정산** 화면에서 해당 서비스/기간으로 금액이 집계되는지 [→ 07-settlement.md]
- [ ] 테스트 데이터는 운영 시작 전 정리(필요 시 구독 강제취소·결제 취소)

---

## 7단계 — 운영 시작 후 모니터링
- [ ] **대시보드**에서 신규 서비스의 구독/매출 추이 관찰 [→ 02-dashboard.md]
- [ ] **결제** 화면에서 실패(FAILED)·미해결(PENDING) 결제가 쌓이지 않는지 주기 확인
- [ ] 이상 동작 시 **감사 로그**에서 누가 무엇을 했는지 추적 [→ 09-audit.md]
- [ ] 점검·사고 시 **전체 설정 → 킬스위치**로 결제서버를 일시 비활성화할 수 있음(어드민은 영향 없음) [→ 10-settings.md]

---

## 자주 하는 실수 / 주의
- **계정보다 서비스를 먼저 만들려 함** → 담당자를 지정할 수 없음. 계정(1단계)을 먼저.
- **키를 메모하지 않고 화면을 벗어남** → 다시 볼 수 없음. 재발급(회전) 필요 + 외부 재설정.
- **허용 IP 누락/오타** → 외부 호출이 전부 차단(401/403). 호출 서버의 *공인* IP인지 확인.
- **요금제를 비활성 상태로 둠** → 구독이 안 됨. ACTIVE인지 확인.
- **취소 정책 미설정** → 단건 결제 취소가 막히거나 수수료가 의도와 다를 수 있음.
- **서비스 삭제가 안 됨** → 구독이나 요금제가 남아 있으면 삭제 불가(정상 동작). 정리 후 삭제하거나 **상태 비활성화**로 대체.
