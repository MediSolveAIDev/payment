# 03. 도메인 — 구독과 결제가 움직이는 규칙

> 목표: 상태머신과 결제 3원칙을 설명할 수 있다. **이 장이 이 시스템의 심장이다.**

## 1. 용어 사전

| 용어 | 의미 | 비고 |
|---|---|---|
| Service | 이 서버를 쓰는 사내 서비스(테넌트) | API키·HMAC 시크릿·허용 IP 보유 |
| Plan | 요금제 — 가격·주기(년/월/주/일)·할인·체험 | **결제 주기는 생성 후 수정 불가** |
| Subscription | (서비스, 사용자)의 구독 1건 | 서비스+사용자당 열린 구독 1개 규칙 |
| Payment | 개별 결제 시도 기록 — 성공/실패 모두 보존 | 금액은 전부 **정수 KRW** |
| external_user_id | 외부 서비스 쪽 사용자 ID 문자열 | 이 서버엔 회원 개념이 없다 |
| customer_key / billing key | 토스 고객키 / 자동결제 키 | 빌링키는 AES 암호화 보관, 평문 저장 금지 |

## 2. 구독 상태머신 — 단일 출처는 `app/services/transitions.py`

```
TRIAL ──▶ ACTIVE ──▶ PAST_DUE ──▶ SUSPENDED ──▶ EXPIRED(종단)
  │          │ ▲        │ ▲           │
  │          │ └────────┘ └─(수동결제)─┴──▶ ACTIVE
  └────┬─────┴─────────────┘
       ▼
   CANCELED ──▶ EXPIRED          (만료 전 재개: CANCELED → ACTIVE | PAST_DUE)
   EXTENDED(운영자 만료일 연장) — 활성과 동등, 새 만료일에 자동갱신   ※ 최근 추가
```

| 상태 | 뜻 | 접근 | 자동결제 |
|---|---|---|---|
| TRIAL | 체험 중(결제 없음) | O | 체험 만료일에 첫 결제 |
| ACTIVE | 정상 | O | 주기마다 갱신 |
| PAST_DUE | 갱신 실패, 재시도 중(미수) | O | 12시간 간격 최대 4회(전역설정) |
| SUSPENDED | 재시도 소진 → 정지 | **X** | 중지 — 수동결제로만 복구 |
| CANCELED | 해지 예약(만료일까지 이용) | O | 중지 — 만료 전 재개 가능 |
| EXTENDED | 운영자가 만료일 수동 연장 | O | 새 만료일에 갱신 |
| EXPIRED | 완전 종료(**종단** — 어떤 전이도 불가) | X | — |

**규칙**: 상태를 바꿀 땐 반드시 `transition(sub, 새상태)`를 호출한다.
허용 전이 테이블(`ALLOWED_TRANSITIONS`)에 없는 전이는 예외로 즉시 터지고,
보편 불변식(EXPIRED/CANCELED→자동결제 중지, SUSPENDED→정지시각 기록,
ACTIVE 복귀→실패 흔적 초기화)이 자동 적용된다. `sub.status = ...` 직접 대입 금지.

## 3. 결제 3원칙 — 절대 깨지 마라

> 이 세 가지가 **이중결제를 막는 전부**다. 결제 코드를 고칠 때마다 자문하라:
> "이 변경이 3원칙을 깨는가?"

1. **PENDING 선커밋** — 토스를 호출하기 *전에* PENDING Payment를 commit한다.
   서버가 죽어도 "돈이 나갔을 수 있다"는 기록이 남아 정산 스윕이 확정할 수 있다.
2. **타임아웃은 실패가 아니다** — 토스 응답을 못 받으면 결과 불명이다. FAILED로
   확정하는 순간, 실제론 승인된 결제 위에 재결제가 얹혀 이중결제가 된다.
   PENDING 유지 → 10분 후 정산 스윕(`reconciliation.py`)이 토스 재조회로 확정.
3. **멱등 order_id** — 갱신 결제는 `(구독, 기간, 시도번호)`로 만든 **결정적**
   order_id를 쓴다. 같은 주문을 재시도하면 토스가 멱등키로 같은 결과를 재생한다.

## 4. 돈 계산 규칙 — 단일 출처는 `app/services/billing_math.py`

- 금액은 전부 **정수(KRW)**. float 금지. 나눗셈은 `//`(내림 — 항상 고객에게 유리).
- **첫 결제액** = 정가 + 첫구독 할인(FREE/정액/정률) — *첫구독에만*.
- **정기 결제액** = 정가 + 상시 할인 — 2회차부터, 그리고 **체험 전환 첫 결제도 이 금액**.
- **취소 수수료** = `금액 × 수수료% // 100`, 환불액 = 금액 − 수수료. (fee+refund==amount 항상 성립)
- 기간 계산(`compute_period_end`)은 relativedelta — 1/31 + 1개월 = 2/28(월말 클램프).

## 5. 그 밖의 핵심 규칙

- **서비스+사용자당 열린 구독 1개** — DB 부분 유니크 인덱스가 최종 심판(EXPIRED만 제외 → 재구독 허용).
- **자동결제 안함(auto_renew=False)** 요금제 — 첫 결제 후 다음 결제를 예약하지 않고 기간 종료 시 만료.
- **첫구독 혜택은 1회** — "결제를 소진한(또는 무료로 활성화된) 과거 구독"이 있으면 재구독은 혜택 없음.
- **접근 판단은 `access_allowed` 하나로** — 외부 서비스는 상태별 분기를 직접 만들지 말 것.
- **요금제·서비스는 구독이 있으면 삭제 불가** (DB FK RESTRICT + 서비스 레이어 검증).
