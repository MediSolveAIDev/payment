# 10. 트러블슈팅 FAQ — 증상으로 찾기

## 서버가 안 뜬다
- **`ENCRYPTION_KEY` 미설정/형식 오류** — 기동 시 가장 먼저 검증한다. `.env.dev`에
  base64 32바이트인지 확인(01장 생성 명령).
- 포트 5433/6380 연결 실패 — `docker compose up -d` 했는지, 다른 프로세스가 점유했는지.

## `alembic upgrade head`가 실패한다
- `Multiple heads`: 두 갈래 리비전 — `uv run alembic heads`로 확인 후 down_revision 체인 수정.
- 이미 수동으로 스키마를 바꾼 DB라면: 마이그레이션과 실제 스키마 차이를 확인하고
  맞춰라(테스트 DB는 그냥 지워도 됨 — 다음 pytest가 재생성).

## 외부 서비스가 401을 받는다
06장의 순서대로: 시계 오차(±300초) → 키 회전 여부(감사 로그 `service.rotate_keys`) →
nonce 재사용 → 서명 본문 불일치. 빠른 검증법: `sample_service`에 같은 키를 넣고
요금제 조회가 되는지 — 되면 키는 정상이고 상대 구현 문제.

## 결제는 됐다는데 DB에 PENDING이다
**정상이다.** 토스 타임아웃 → 결과불명 → 10분 후 정산 스윕이 토스 재조회로 확정한다.
절대 수동으로 FAILED 처리하지 마라(07장). 15분 넘게 안 풀리면 로그 + 토스 콘솔 대조.

## 어드민에서 어떤 항목이 404가 난다 (분명히 존재하는데)
의도된 동작일 수 있다 — SERVICE_MANAGER는 담당 외 서비스 리소스를 **404로 숨긴다**
(존재 비노출). SYSTEM_ADMIN 계정으로 보이면 권한 스코프 문제다.

## 테스트가 수십 개씩 무더기로 실패한다
다른 pytest가 동시에 돌고 있는지부터 의심(09장) — **단독 재실행이 1순위 진단**.
단독에서도 실패하면 그때가 진짜 버그다.

## manual.html을 고쳤는데 다음 빌드에서 사라졌다
`manual.html`/`index.html`은 **생성물**이다. 원본 마크다운(또는 build_html.py)을 고치고
재빌드하라. (dev_manual·new_manual 동일)

## 샘플 쇼핑몰이 "구독서버에 연결할 수 없습니다"라고 한다
구독서버(:8000)가 떠 있는지 → 샘플의 서비스 키가 맞는지(어드민 키 화면과 대조) →
서비스 허용 IP에 `127.0.0.1`이 있는지.

## 갱신이 안 도는 것 같다
- `.env`의 `SCHEDULER_ENABLED`/`SCHEDULER_INTERVAL_MINUTES` 확인.
- 로그에 `renewal batch skipped — 다른 인스턴스가 실행 중`이 계속 찍히면:
  죽은 프로세스의 전역 락이 남은 것 — Redis에서 `lock:scheduler:renewals` TTL(최대 240초) 후 자연 해소.
- 수동 1회 실행으로 확인: 테스트의 `run_renewals(app)` 패턴 참고.

## 이메일이 안 나간다
`GMAIL_ID`/`GMAIL_PW`(앱 비밀번호) 미설정이면 **콘솔 출력 모드**가 기본 — 로그에서
메일 내용을 확인할 수 있다. 실제 발송은 두 값을 설정해야 한다.
