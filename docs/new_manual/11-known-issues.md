## 1. 수정 권장 — 아직 미적용 (로직 검증 High)

손대기 전에 반드시 원문 리포트의 재현 시나리오를 읽을 것.

| # | 문제 | 한 줄 요약 |
|---|---|---|
| H-1 | 수동 결제 이중 청구 | 수동 결제가 구독 락 없이 매번 새 멱등키로 토스 호출 — 배치와 겹치거나 더블클릭이면 2회 청구 |
| H-2 | 승인된 돈 영구 미기록 | 정산 스윕이 수동 결제 PENDING을 "배치가 처리할 것"으로 오인해 영원히 건너뜀 |
| H-3 | 재시도 중 가격 변경 시 금액 불일치 | PENDING 재사용 시 청구액을 재계산하지만 DB 기록액은 안 바꿈 |
| H-4 | 100% 상시할인 = 구독 자동 파괴 | 0원 갱신 청구 가드 없음 — 토스 거절 반복 → 정지 → 만료 |

## 2. 정책 결정이 먼저 필요한 것 (코드보다 의사결정)

| 주제 | 현재 동작 | 결정할 것 |
|---|---|---|
| 환불 월 귀속 | 원결제 월에 소급 반영(`canceled_at` 미사용) | 취소 시점 월로 집계할지 |
| 월말 앵커 | 1/31 가입 → 영원히 28일로 드리프트 | 가입일 유지 정책으로 바꿀지, 현행 문서화할지 |
| 체험 후 첫구독 혜택 | 체험만 쓴 사용자는 할인 영구 상실 | 의도(악용 방지)인지 확인·문서화 |
| 집계 시간대 | 집계는 UTC 경계, 표시는 KST(00~09시 어긋남) | KST 경계로 통일할지 |
| 연장(EXTENDED)×비자동갱신 | 연장하면 자동결제가 예약됨 | "이용만 연장"인지 "자동결제 재개"인지 |

## 3. 알고 보류한 것 (사유 포함 — 워크로그에 상세)

- 보안: 토스 웹훅 서명(토스 미제공 — IP+재조회로 방어), 로그인 폼 CSRF(사내 어드민·rate limit 보완),
  dev 비밀번호 프리필(배포 체크리스트로 대응)
- 구조: 대시보드 표시 로직 분리·registry 이름 변경·conftest 리포트 분리 (이득 대비 보류)

## 4. 진행 중 작업 — EXTENDED(연장처리) 상태

워킹트리에서 진행 중(2026-06 기준). enum·전이표·배치·화면까지 반영도가 높고 테스트
통과 상태지만, **잔여 3건**: ① 비자동갱신 요금제 연장 정책(위 표), ② 외부 API
스키마 설명·샘플·dev_manual 미반영, ③ 연장 시 기존 갱신 PENDING 고아화(H-2와 함께 해소).
이 작업을 이어받는다면 08장 레시피 D의 체크리스트로 누락을 점검하라.
