CHAPTER 3

결제·환불

정기 구독 갱신과 단건 결제를 포함한 모든 거래를 확인하고, 환불(취소)을 처리합니다.

이 화면은 '영수증 보관함'입니다. 구독으로 자동 빠져나간 돈이든, 손님이 1회성으로 산 것이든 모든 결제가 한 줄씩 쌓입니다. 환불해 줘야 할 일이 생기면 여기서 해당 영수증을 찾아 처리합니다.

총매출은 받은 돈 전체, 순매출은 환불을 빼고 실제로 손에 남은 돈입니다.

3.1결제 내역 목록

개별 결제 건의 성공 여부와 금액을 확인합니다. 표의 주요 열은 다음과 같습니다.

열의미
승인 시각결제가 최종 완료된 시각
사용자결제한 사용자 ID
주문번호고유 주문 번호. PG(토스) 문의 시 가장 중요한 기준
유형정기(Renewal) / 단건(One-off)
상태결제 결과(아래 참고)
총매출 / 환불 / 순매출결제 금액, 환불액(빨강), 그리고 둘의 차액인 최종 금액

결제 상태

상태의미
DONE 완료정상 승인됨
PENDING 대기처리 중(보통 잠시 후 확정)
FAILED 실패결제 중 오류 발생
CANCELED 취소환불(취소) 처리됨

3.2결제 상세 정보

특정 결제를 클릭하면 더 자세한 데이터를 볼 수 있습니다.

  • 상품명 — 결제 시 표시된 상품명. 일반(단건) 결제는 요청 시 전달한 주문명, 구독 결제는 요금제명이 표시됩니다(과거 데이터는 -)
  • 실패 코드·메시지 — 실패 시 카드사가 보낸 정확한 사유(잔액부족, 카드정지 등)
  • 토스 결제키(Payment Key) — 토스 관리자 페이지에서 해당 거래를 찾을 때 쓰는 키
  • 승인 시각 — 카드사 승인이 완료된 초 단위 시각
  • 원본 응답(Raw Response) — PG가 보낸 데이터 전체. 평상시엔 무시해도 되며 장애 분석 시 참고

3.3환불(취소) 처리

⚠️

환불은 단건(One-off) 결제에 대해 처리합니다. 환불 가능 여부와 수수료는 서비스의 취소 정책(취소 허용 여부·수수료율)에 따릅니다. 취소 정책은 시스템 관리자가 서비스 설정에서 지정합니다.

  1. 예상 환불액 확인

    결제 상세 화면에 서비스 취소 정책에 따른 예상 수수료·환불 금액이 미리 표시됩니다.

  2. 결제 취소 실행

    결제 취소 버튼을 누르면 실시간으로 PG(토스)에 취소 요청이 전달됩니다.

  3. 결과 확인

    완료되면 상태가 CANCELED로 바뀌고, 실제 환불액과 취소 수수료가 기록됩니다.

⛔

이미 취소된 결제는 되돌릴 수 없습니다. 환불 전 금액과 대상 사용자를 반드시 확인하세요.

취소 수수료는 환불할 때 떼는 '위약금' 같은 것입니다. 정책에 10%로 정해 두면, 1만 원을 환불할 때 1천 원은 우리가 갖고 9천 원만 손님에게 돌려줍니다. 화면에는 버튼을 누르기 전에 "이만큼 돌려주게 됩니다"가 미리 표시되니 금액을 확인하고 진행하면 됩니다.

ℹ️

예: 취소 수수료율이 10%라면, 10,000원 결제를 취소할 때 1,000원이 수수료로 차감되고 9,000원이 환불됩니다.

3.4영수증(매출전표)

결제 상세 화면에서 고객에게 전달할 수 있는 매출전표(영수증) 링크를 제공합니다. 고객이 영수증 발행을 요청하면 이 링크를 복사해 전달하면 됩니다(카드사 공식 영수증).

💡

구독 결제와 환불 — 정기 구독 결제 자체의 취소가 필요하면 먼저 구독 관리에서 구독을 정리하고, 개별 결제 건 환불은 이 화면에서 정책에 따라 처리합니다.
