CHAPTER 11

자주 묻는 질문 (FAQ)

자주 묻는 질문과 문제 해결을 모았습니다. 구독·결제 상태값과 API 에러 코드는 12. 상태·코드 레퍼런스를 보세요.

11.1자주 묻는 질문

구독·요금제

한 사용자가 요금제 2개를 동시에 구독할 수 있나요?

아니요. 서비스 + 사용자당 구독은 1개입니다. 이미 열린 구독이 있으면 새 구독 생성이 409 CONFLICT로 거부됩니다. 요금제를 바꾸려면 기존 구독을 정리한 뒤 다시 생성하세요.

결제 주기를 나중에 바꿀 수 있나요?

아니요. 결제 주기는 요금제 생성 시에만 설정되고 이후 수정할 수 없습니다. 다른 주기가 필요하면 새 요금제를 만드세요.

요금제/서비스를 삭제하려는데 안 돼요.

구독이 남아 있으면 삭제할 수 없습니다. 요금제는 보관(Archive) 처리하면 신규 구독만 막고 기존 구독은 유지됩니다.

'무료'와 '체험'은 무엇이 다른가요?

무료(FREE)는 첫 회차를 0원으로 처리하며 구독이 바로 시작됩니다. 체험(Trial)은 결제 없이 일정 기간 이용하다가 체험 종료 시 첫 결제가 발생합니다. 체험은 요금제에서 체험 제공을 켰을 때만 가능합니다.

첫 구독 결제가 실패하면 구독·대시보드에 어떻게 반영되나요?

첫 결제가 확정 실패(카드 거절 등)하면 그 구독은 구독·결제 테이블에 저장되지 않고 감사 로그에만 남습니다. 성사되지 않은 구독을 남기면 대시보드 "신규 구독" 수가 부풀려져 데이터 의미가 틀어지기 때문입니다(실제 결제된 구독만 집계). 재시도하면 첫구독 혜택은 그대로 적용됩니다. 자세히는 2.7. (타임아웃은 예외 — PENDING으로 보존 후 정산 스윕이 확정)

결제·환불

구독을 강제 취소하면 환불도 되나요?

아니요. 강제 취소는 환불을 포함하지 않습니다. 환불이 필요하면 결제·환불 메뉴에서 해당 결제 건을 정책에 따라 별도로 취소하세요.

고객 카드가 정리됐다는데 다음 결제까지 기다려야 하나요?

아니요. 구독 상세에서 즉시 재결제로 지금 바로 결제를 시도할 수 있습니다. 성공하면 구독이 즉시 ACTIVE로 복구됩니다.

환불이 안 됩니다.

서비스의 취소 허용이 꺼져 있거나, 대상이 단건 결제가 아니거나, 이미 취소된 건일 수 있습니다. 취소 정책은 시스템 관리자가 서비스 설정에서 변경합니다.

로그인·계정

비밀번호 설정 링크가 만료됐어요.

링크는 48시간·1회용입니다. 시스템 관리자에게 비밀번호 재설정으로 새 링크 발송을 요청하세요.

로그인이 잠겼어요(LOCKED).

비밀번호 5회 이상 실패 시 자동 잠금됩니다. 시스템 관리자가 잠금을 해제해야 합니다.

다른 서비스 데이터가 안 보여요.

정상입니다. 서비스 담당자는 본인 담당 서비스만 볼 수 있습니다.

API 연동(개발)

계속 401(UNAUTHORIZED)이 납니다.

대부분 ① 서명에 쓴 본문과 전송 본문의 바이트가 다름, ② 서버·클라이언트 시계 차이(±5분 초과), ③ nonce 재사용, ④ 잘못된 HMAC 시크릿 중 하나입니다. 본문을 한 번만 직렬화해 서명·전송에 동일하게 쓰세요.

403(FORBIDDEN)이 납니다.

호출 서버의 공인 IP가 허용 IP 목록에 없습니다. 운영/개발 서버 IP를 등록 요청하세요.

429(RATE_LIMITED)가 납니다.

요청 한도를 넘었습니다. 잠시 대기 후 재시도(backoff)하세요. 결제성 API는 한도가 더 엄격합니다.

503(SERVER_DISABLED)이 납니다.

결제서버 전체가 일시 중단(킬스위치)된 상태입니다. 긴급 점검 등 운영상 조치이며, 시스템 관리자에게 문의하세요.
