CHAPTER 12

상태·코드 레퍼런스

구독·결제 상태값과 API 에러 코드를 한곳에 모은 레퍼런스입니다. 자주 묻는 질문은 11. FAQ를 보세요.

12.1구독 상태 레퍼런스

상태의미접근자동결제
TRIAL무료 체험 중(첫 결제 전)허용체험 종료 시 첫 결제
ACTIVE정상 이용 중허용만료일에 갱신
PAST_DUE결제 실패, 자동 재시도 중허용재시도 진행
EXTENDED관리자가 만료일 수동 연장허용새 만료일에 갱신
CANCELED해지 예약(만료일까지 유지)허용없음(만료 시 종료)
SUSPENDED반복 실패로 정지차단수동 결제로 복구
EXPIRED완전 종료(최종)차단없음

access_allowed는 TRIAL·ACTIVE·PAST_DUE·CANCELED·EXTENDED에서 true, SUSPENDED·EXPIRED에서 false입니다.

12.2결제 상태 레퍼런스

상태의미
PENDING처리 중(정산 스윕에서 확정)
DONE정상 승인 완료
FAILED결제 실패(failure_code/message 확인)
CANCELED취소(환불) 처리됨

결제 회차 구분: FIRST(첫 결제) · RENEWAL(정기 갱신) · RETRY(재시도) · ONE_OFF(단건).

12.3API 에러 코드 레퍼런스

HTTPcode의미
401UNAUTHORIZED인증 헤더/서명 누락·불일치
403FORBIDDEN허용되지 않은 IP
402PAYMENT_FAILED토스 결제 승인/자동결제 실패
404NOT_FOUND리소스 없음
409CONFLICT구독 1개 규칙 위반 등 상태 충돌
422VALIDATION_ERROR형식 오류·비즈니스 규칙 위반
429RATE_LIMITED요청 한도 초과
503SERVER_DISABLED결제서버 비활성화(킬스위치)
