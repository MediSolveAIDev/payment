개발자 인수인계 매뉴얼 (dev_manual)

🍎

쉽게 말하면: 이 서버는 사내 여러 앱이 '구독·결제'를 각자 만들지 않고 한곳에 맡기는 공용 결제 창구입니다. 건물의 모든 가게가 1층 공용 카운터에서 계산하는 것과 같습니다. 외부 앱은 API로 "이 사용자 구독시켜줘"라고 요청만 하고, 실제 토스 결제·자동 갱신·정산은 이 서버가 대신 처리합니다.

처음 인수받는 분께: 읽는 순서가 있는 학습 경로형 가이드 인수인계 가이드(new_manual)부터 시작하세요 — 첫날 셋업 → 큰그림 → 도메인 규칙 → 기능 추가 레시피 순서로 안내합니다. 이 dev_manual은 기능별 상세 레퍼런스(백과사전)입니다.

이 문서는 개발 초보자가 인수인계만으로 바로 유지보수할 수 있도록, 기능별로 "요청 → 거치는 파일/코드 → 사용하는 DB 테이블 → 응답"을 끝까지 추적해 설명합니다.

이 시스템은 무엇인가

사내 여러 서비스가 공통으로 쓰는 구독·결제 API 서버입니다. - 외부 서비스는 API(/api/v1/...)로 구독·결제를 요청합니다(HMAC 서명 인증). - 운영자는 어드민 콘솔(/admin/...)에서 서비스·요금제·구독·결제·정산을 관리합니다(로그인 세션). - 결제는 토스페이먼츠로 처리합니다. - 기술: FastAPI · PostgreSQL(SQLAlchemy 2 async) · Redis · Jinja2/htmx(어드민) · APScheduler(배치) · TossPayments.

읽는 순서

처음 인수인계라면 01 → 02 → 03을 먼저 읽고(공통 기반), 그다음 담당 기능 문서를 봅니다.

화면을 직접 쓰는 운영자/유지보수 담당자를 위한 페이지별 설명입니다. - 어드민 콘솔 페이지 매뉴얼 (목차) — 로그인·대시보드·서비스·요금제·구독·결제·정산·계정·감사로그·전체설정

각 기능 문서의 공통 구성

  1. 한 줄 요약 — 이 기능이 무엇을 하는가
  2. 언제 실행되나 — 트리거(외부 API 호출 / 어드민 화면 / 스케줄러)
  3. 요청 진입점 — HTTP 메서드·경로·라우트 함수(file:line)
  4. 단계별 처리 흐름 — 요청부터 응답까지 거치는 파일/함수와 그 역할, 각 단계의 DB 작업
  5. 사용하는 DB 테이블·컬럼 — 읽기/쓰기 구분
  6. 상태 전이 — (해당 시) 상태 머신
  7. 예외·엣지 케이스 / 에러 응답
  8. 관련 테스트 — 동작을 확인·재현할 수 있는 테스트 위치
  9. 유지보수 팁 — "이걸 바꾸려면 어느 파일을 고친다", 흔한 디버깅 포인트

HTML 버전 (한 파일로 보기)

브라우저로 한 번에 탐색하려면 manual.html 을 엽니다(사이드바 목차·검색·문서 간 링크 내장, 오프라인/인쇄 가능). 이 매뉴얼 전체(기능 16 + 어드민 11 + 온보딩)를 한 페이지로 묶은 인수인계용 문서입니다. - 갱신(마크다운 수정 후 재생성): uv run --with markdown python docs/dev_manual/build_html.py
