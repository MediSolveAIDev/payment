어드민 콘솔 페이지 매뉴얼

💬

어드민 콘솔(/admin)의 화면별 사용 가이드 + 개발 참조입니다. - 운영자: 각 화면의 목적·접근 경로·할 수 있는 동작(버튼/폼)과 결과·주의사항을 봅니다. - 개발자: 각 화면의 라우트 함수·템플릿·htmx 동작·호출 서비스·수정 방법을 봅니다.

기능의 내부 처리 흐름(요청→코드→DB)은 상위 dev_manual의 기능별 문서를 함께 참고하세요.

화면 목록

  • 01. 로그인·접근 (로그인 / 비밀번호 설정 / 로그아웃 / IP 제한)
  • 02. 대시보드
  • 03. 서비스 (목록·등록·상세·키발급/회전·담당자·IP·취소정책·상태·삭제)
  • 04. 요금제 (목록·생성·수정·활성/비활성·삭제)
  • 05. 구독 (목록·상세·강제취소)
  • 06. 결제 (목록·상세·취소·실패코드 툴팁)
  • 07. 정산
  • 08. 계정 (목록·생성·상세·수정·비활성·삭제·비번재설정·서비스 배정)
  • 09. 감사 로그 (조회·검색·정리·엑셀)
  • 10. 전체 설정 (재시도·어드민 IP·킬스위치)

운영 워크플로우

  • 신규 서비스 온보딩 체크리스트 (운영자용) — 계정 생성→서비스 등록→키 전달→요금제→연동→검증 순서

공통 개념 (모든 화면에 적용)

레이아웃·내비게이션

  • 좌측 사이드바(LNB) + 상단 탑바 + 본문. 템플릿 골격은 app/admin/templates/base.html.
  • 메뉴 활성 표시·아이콘은 base.html의 LNB. 디자인 토큰/스타일은 app/static/admin.css(Centurion Suite).

권한·스코프 (중요)

  • SYSTEM_ADMIN: 전체 서비스·계정·설정 접근.
  • SERVICE_MANAGER: 담당 서비스 범위로만 조회/관리(타 서비스는 보이지 않거나 404).
  • 라우트는 require_admin/require_any/require_manager + service_scope(ctx)로 제한합니다(상세: ../03-auth-and-security.md).
  • 접근 권한이 없으면 목록에서 빠지거나 404(존재 노출 방지)로 응답합니다.

공통 UI 패턴

  • 목록 화면: 검색(q)·정렬(컬럼 헤더 클릭)·페이지네이션·필터·엑셀 다운로드(*/export.xlsx). 공통 매크로 app/admin/templates/_list.html, 파라미터 처리 app/admin/pagination.py.
  • 저장 완료 모달: DB에 저장/수정하는 동작 성공 시 "저장되었습니다" 모달이 뜹니다(saved_redirect() + showSavedModal, htmx는 HX-Trigger: showSaved). 구현 app/static/admin.js.
  • 확인 모달: 위험 동작(삭제·비활성화·킬스위치 등)은 data-confirm으로 확인 모달을 띄운 뒤 진행합니다. 모달은 버튼으로만 닫힙니다.
  • 토스트: 처리 결과/오류 안내(상단). 에러는 ?error= 리다이렉트로 전달되어 토스트로 표시됩니다.
  • CSRF: 모든 상태변경(POST) 폼에는 csrf_token hidden이 들어가고 서버가 검증합니다.
  • 캐시버스팅: CSS/JS는 ?v=mtime로 로드되어 변경 시 브라우저에 즉시 반영됩니다.
  • 시각 표시: 저장은 UTC, 화면은 KST({{ dt|kst }}).

각 화면 문서의 구성

  1. 이 화면은 무엇인가 / 접근 경로·권한
  2. 화면 구성 — 무엇이 보이나(목록 컬럼·상세 항목·탭 등)
  3. 할 수 있는 동작 — 버튼/폼별: 무엇을 하면 → 어떤 결과(확인/저장 모달 포함)
  4. 개발 참조 — 라우트 함수(file:line)·템플릿·htmx·호출 서비스·관련 기능 문서 링크
  5. 주의사항 / 자주 하는 실수
