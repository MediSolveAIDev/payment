# 구독/결제 서버 코드 분석 매뉴얼

이 폴더는 구독/결제 서버의 코드를 **체계적으로 분석**한 문서 모음이다.
"기능(프로세스)별로 무엇을, 어떤 코드를 통해, 어떻게 처리하는가"를 설명한다.

## 읽는 순서

1. **[00-overview.md](00-overview.md) — 전체 구성 개요** ✅
   시스템 목적, 기술 스택, 계층 구조, 디렉터리 맵, 두 진입 평면(외부 API/Admin),
   데이터 모델·상태 머신, 공통 처리 흐름, 횡단 관심사. **먼저 읽을 것.**

2. 기능별 프로세스 문서 **01~12 (전체 완료 ✅)** — 아래 표 참고.
   기초(01 서비스·02 계정·03 요금제) → 핵심 결제(04 구독생성·05 갱신·06 관리·07 정합성·11 단건결제)
   → 인증(08) → 집계·화면(09) → 감사(10) → 공통 기능(12 엑셀 다운로드) 순으로 읽으면
   의존 순서대로 이해할 수 있다.

## 문서 구성 규칙

각 기능 문서는 동일한 틀을 따른다:

1. **프로세스 정의** — 이 기능이 무엇을 보장/수행하는가 (입력·출력·규칙)
2. **관여 코드 맵** — 라우트 / 서비스 계층 / 모델 / 외부연동 중 어디가 쓰이나
3. **처리 흐름** — 단계별 호출 순서 + 상태 전이 + 트랜잭션/감사로그 지점
4. **예외·엣지 케이스** — 실패·동시성·멱등성 처리
5. **관련 테스트** — 동작을 고정하는 테스트 위치
