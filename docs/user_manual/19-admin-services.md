# 서비스 관리 (목록·등록·키 발급)

> 쉽게 말하면, **"어떤 사내 앱이 이 결제 서버를 쓸지" 를 등록·관리**하는 화면입니다. 서비스를 등록해야 그 서비스용 API 키·토스 시크릿 키·요금제·구독이 생깁니다. 전체 흐름에서 **가장 먼저** 하는 일입니다([0. 개요](00-overview.md)의 1~5 준비 단계).

> 대상: **시스템 관리자(SYSTEM_ADMIN) 전용** 화면입니다. 서비스 담당자(SERVICE_MANAGER)에게는 이 메뉴가 보이지 않습니다.

> 함께 보기: [관리자 콘솔 사용하기](01-admin-console.md) · [계정 관리](06-admin-accounts.md) · [요금제 관리](04-admin-plan.md) · 화면별 기술 레퍼런스는 [어드민 화면별 설명](16-admin-screens.md)의 16.4.

---

## 시작 전에 — 담당자 계정을 먼저

서비스 등록 화면은 담당자를 **목록에서 선택**하는 방식입니다. 그래서 **담당자 계정(SERVICE_MANAGER)을 먼저 만들어야** 서비스에 담당자를 지정할 수 있습니다. 계정 생성은 [계정 관리](06-admin-accounts.md)를 참고하세요.

---

## 1. 서비스 목록

왼쪽 메뉴에서 **서비스**를 누르면 등록된 서비스 목록이 열립니다.

- **검색**: 서비스명 또는 담당자(이메일) 일부로 검색
- **상태 필터**: <span class="pill ok">활성(ACTIVE)</span> / <span class="pill no">비활성(INACTIVE)</span> / 전체
- **컬럼**: 서비스명, 담당자, 허용 IP, 상태
- 서비스명을 누르면 **서비스 상세**로 이동합니다.
- 오른쪽 위 **[서비스 등록]** 버튼으로 새 서비스를 만듭니다.

> 비활성(INACTIVE) 서비스는 API 키 인증이 막혀 외부 호출이 모두 거부됩니다(상세에서 활성/비활성을 토글).

---

## 2. 서비스 등록

**서비스 → 서비스 등록**(`/admin/services/new`)에서 입력합니다.

| 항목 | 필수 | 설명 |
|------|------|------|
| **서비스명** | 필수 | 콘솔에 표시될 이름(중복 불가) |
| **담당자 계정** | 필수 | 미리 만든 SERVICE_MANAGER 계정을 **복수 선택** |
| **대표 계정** | 필수 | 결제 실패·갱신 등 **알림 메일 수신처**. 체크 목록에 없어도 자동으로 담당자에 포함됨 |
| **허용 IP** | 선택 | API를 호출할 서버의 공인 IP(IPv4, 한 줄에 하나). 비우면 IP 제한 없음 |
| **토스 시크릿 키** | 선택 | 그 서비스의 토스 시크릿 키. **AES 암호화 저장**, 저장 후 다시 표시되지 않음(쓰기 전용). 지금 비워두고 나중에 상세에서 설정/교체 가능 |
| **일반결제 취소 정책** | — | 단건(일반) 결제 취소 허용 여부 + 취소 수수료율(0~100%) |

> 중요: 결제가 동작하려면 **그 서비스의 토스 시크릿 키가 등록**돼 있어야 합니다(서비스마다 다름). 미등록 서비스는 결제·구독 첫 결제·자동연장이 `TOSS_KEY_NOT_CONFIGURED`(422)로 거부됩니다. 자세한 동작은 [어드민 화면별 설명](16-admin-screens.md) 16.4 참고. (전역 `TOSS_SECRET_KEY` 환경변수는 제거됨)

---

## 3. 키 발급·전달 (등록 직후, 1회성 · 중요)

등록을 누르면 **키 발급 화면**으로 이동합니다.

- **API 키**와 **HMAC 시크릿**이 화면에 **단 한 번만** 표시됩니다(키는 암호화 저장되므로 평문을 볼 수 있는 유일한 기회).
- 두 값을 **복사**해 **안전한 채널**(사내 비밀 저장소 등)로 서비스 담당 개발자에게 전달하세요.
- 화면을 벗어나면 평문 키를 다시 볼 수 없습니다.
- 분실·유출 시: 상세에서 **키 재발급(회전)** — 새 키가 즉시 발급되고 **이전 키는 무효화**되므로, 외부 서비스가 새 키로 다시 설정해야 합니다.

> API 키·HMAC 시크릿의 용도(요청 서명)는 [서비스 연동 API](11-service-api.md)를 참고하세요. 토스 시크릿 키는 **서버가 토스에 결제 요청**할 때 쓰는 별도 값이며, 외부 서비스의 client key와는 다릅니다(client key는 서비스 프론트가 자체 사용).

---

## 4. 서비스 상세 — 설정·관리

서비스명을 누르면 상세 화면이 열리고, 탭과 설정으로 관리합니다.

- **탭**: 요금제 · 구독 · 등록 카드 · 단건 결제 · 이벤트(이 서비스 관련 감사 로그)
- **설정 변경**:
  - **허용 IP** 갱신
  - **일반결제 취소 정책**(허용 여부 / 수수료율)
  - **알림 수신 URL**(아웃고잉 웹훅 — 비우면 미발송) → [서비스 알림 기능](15-feature-notifications.md)
  - **토스 시크릿 키** 설정/교체(쓰기 전용, AES 암호화). 빈 값으로 저장하면 변경 없음. 상세에는 "설정됨/미설정"만 표시
  - **상태**(ACTIVE/INACTIVE) 토글
  - **키 재발급(회전)**
  - **담당자 관리**(추가/제거, 대표 지정 — 대표는 해제 불가)
- **삭제**: 구독이 있으면 삭제가 거부됩니다(먼저 구독 정리 필요).

> 토스 시크릿 키 설정/변경 등 이 화면의 주요 동작은 모두 **감사 로그**에 남습니다(시크릿 값 자체는 기록하지 않음). → [감사 로그](08-admin-audit.md)
