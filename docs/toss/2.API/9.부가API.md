## 지급대행

지급대행은 오픈마켓에서 발생한 매출을 토스페이먼츠가 오픈마켓을 대신하여 셀러(입점 판매자)에게 지급하는 서비스입니다. 직접 수많은 셀러에게 정산할 필요 없이, 정산 업무를 토스페이먼츠한테 맡기고 자금 흐름을 더 효율적으로 관리할 수 있어요.

토스페이먼츠와 직접 계약하지 않은 셀러에게 토스페이먼츠가 정산금을 보내야 되기 때문에 [KYC](/resources/glossary/kyc) 등 리스크 검토 절차가 있습니다. 그리고 연동 측면에서도 더 강화된 보안을 요구합니다. Request Body가 있는 **지급대행 POST 요청은 모두 [`ENCRYPTION` 보안](/guides/v2/payouts#encryption-보안)을 사용합니다.** `ENCRYPTION` 보안은 토스페이먼츠에서 발급하는 보안 키를 사용해서 API의 Request Body를 JWE로 암호화하는 방식입니다. 자세한 암호화 방법은 [지급대행 가이드](/guides/v2/payouts)에서 확인해주세요.

### Balance 객체

오픈마켓 쇼핑몰에서 셀러에게 지급할 수 있는 정산 잔액 정보입니다.

#### 객체 상세

- **pendingAmount** 셀러에게 지급할 수 없는 잔액 정보입니다. 발생한 매출이 정산 주기 2일 전에 `pendingAmount` 금액으로 업데이트됩니다. 토스페이먼츠 정산 주기가 지난 금액은 `availableAmount`로 전환됩니다.
  - **currency** 통화입니다. 현재는 `KRW`만 지원합니다.
  - **value** 잔액입니다.

- **availableAmount** 셀러에게 지급할 수 있는 잔액 정보입니다.
  - **currency** 통화입니다. 현재는 `KRW`만 지원합니다.
  - **value** 잔액입니다.

method: GET
path: /v2/balances

### 잔액 조회

셀러에게 지급할 수 있는 정산 금액을 조회합니다.

#### Response

잔액 조회에 성공하면 [Balance 객체](#balance-객체)가 돌아옵니다.

잔액 조회에 실패하면 [에러 객체](/reference/using-api/req-res#api-v2-에러-객체)가 돌아옵니다.

### Seller 객체

오픈마켓에 입점되어 있는 셀러의 사업자, 계좌 정보입니다.

#### 객체 상세

- **id** 토스페이먼츠에서 발급하는 셀러의 고유 식별자입니다. 셀러 조회, 삭제, 지급대행 요청에 사용됩니다. 셀러의 상태 및 지급대행 상태가 바뀌어도 `id`는 바뀌지 않고, 삭제된 `id`는 다시 발급되지 않습니다. 최대 길이는 35자입니다.

- **refSellerId** `nullable` 오픈마켓 상점에서 직접 등록하는 셀러의 고유 식별자입니다. 연동 중에 참고할 수 있는 값입니다. 등록 이후에는 수정할 수 없습니다. 허용된 길이는 7-20자 입니다.

- **businessType** 사업자 유형입니다. `INDIVIDUAL`(개인), `INDIVIDUAL_BUSINESS` (개인사업자), `CORPORATE`(법인사업자) 중 하나입니다.

- **company** `nullable` 법인사업자 또는 개인사업자 셀러의 정보입니다.
  - **name** 사업자명입니다
  - **representativeName** 대표자명입니다.
  - **businessRegistrationNumber** 사업자번호입니다
  - **email** 사업자 이메일입니다. 최대 길이는 100자입니다. 해당 이메일로 KYC 심사 안내가 전송됩니다.
  - **phone** 사업자 전화번호입니다.

- **individual** `nullable` 개인 셀러의 정보입니다.
  - **name** 개인의 이름입니다.
  - **email** 개인 이메일입니다. 최대 길이는 100자입니다. 해당 이메일로 KYC 심사 안내가 전송됩니다.
  - **phone** 개인 전화번호입니다.

- **status** 셀러 상태입니다. 셀러 상태가 바뀌면 [`seller.changed`](/reference/using-api/webhook-events#sellerchanged)웹훅이 발송됩니다.

  \- `APPROVAL_REQUIRED`: 지급대행이 불가능한 상태입니다. 개인 및 개인사업자 셀러 등록 직후의 상태이며, 본인인증이 필요합니다.

  \- `PARTIALLY_APPROVED`: 일주일 동안 1천만원까지 지급대행이 가능한 상태입니다. 등록 직후의 법인사업자 셀러 또는 본인인증을 완료한 개인 및 개인사업자 셀러의 상태입니다.

  \- `KYC_REQUIRED`: 지급대행이 불가능한 상태입니다. 일주일 동안 1천만원을 초과하는 금액을 지급 요청하면 셀러는 해당 상태로 변경됩니다. 셀러가 KYC 심사를 완료해야 합니다.

  \- `APPROVED`: 금액 제한 없이 지급대행이 가능한 상태입니다. KYC 심사가 정상적으로 완료된 셀러의 상태입니다.

- **account** 셀러가 정산금을 지급받을 계좌 정보입니다.
  - **bankCode** 은행 세 자리 코드입니다. [은행 코드](/codes/org-codes#은행-코드)와 [증권사 코드](/codes/org-codes#증권사-코드)를 참고하세요.
  - **accountNumber** 계좌번호입니다. 최대 길이는 20자입니다.
  - **holderName** 계좌 예금주입니다. 최대 길이는 공백을 포함해 한영 구분 없이 50자입니다.

- **metadata** `nullable` 셀러와 관련된 추가 정보를 key-value 쌍으로 담고 있는 객체입니다. 최대 5개의 키-값(key-value) 쌍입니다. 키는 `[` , `]` 를 사용하지 않는 최대 40자의 문자열, 값은 최대 2000자의 문자열입니다.

method: POST
path: /v2/sellers

### 셀러 등록

오픈마켓에 입점해있는 셀러의 정보를 토스페이먼츠에 등록합니다. 셀러의 사업자번호, 계좌 유효성를 검증합니다.

셀러 등록은 지급대행 서비스의 POST 요청입니다. [`ENCRYPTION` 보안](/guides/v2/payouts#encryption-보안)으로 요청을 보내주세요.

#### Request Body 파라미터

- **refSellerId** 오픈마켓 상점에서 발급하는 셀러의 고유 식별자입니다. 연동 중에 참고할 수 있는 값입니다. 등록 이후에는 수정할 수 없습니다. 허용된 길이는 7-20자 입니다.

- **businessType** 사업자 유형입니다. `INDIVIDUAL`(개인), `INDIVIDUAL_BUSINESS`(개인사업자), `CORPORATE`(법인사업자) 중 하나입니다.

- **company** 법인사업자 또는 개인사업자 셀러 정보입니다. `businessType`이 `INDIVIDUAL_BUSINESS` 또는 `CORPORATE`이면 필수입니다.
  - **name** 사업자명입니다.
  - **representativeName** 대표자명입니다.
  - **businessRegistrationNumber** 사업자번호입니다. `-` 없이 10자리 사업자번호 숫자만 입력해주세요.
  - **email** 이메일입니다
  - **phone** 사업자 전화번호입니다. `-` 없이 숫자로만 구성된 최소 8자의 문자열입니다.

- **individual** 개인 셀러 정보입니다. `businessType`이 `INDIVIDUAL`이면 필수입니다.
  - **name** 개인의 이름입니다.
  - **email** 이메일입니다
  - **phone** 전화번호입니다. `-` 없이 숫자로만 구성된 최소 8자의 문자열입니다.

- **account** 셀러가 정산 금액을 지급받을 계좌 정보입니다.
  - **bankCode** 계좌의 은행 코드입니다. [은행 코드](/codes/org-codes#은행-코드)와 [증권사 코드](/codes/org-codes#증권사-코드)를 참고해서 두 자리 코드 또는 세 자리 코드를 사용해주세요.
  - **accountNumber** 계좌번호입니다. `-` 없이 숫자만 입력해주세요. 최대 길이는 14자입니다.
  - **holderName** 계좌 예금주입니다. 최대 길이는 공백을 포함해 한영 구분 없이 50자입니다.

- **metadata** 셀러와 관련된 추가 정보를 key-value 쌍으로 담고 있는 JSON 객체입니다. 최대 5개의 키-값(key-value) 쌍입니다. 키는 `[` , `]` 를 사용하지 않는 최대 40자의 문자열, 값은 최대 2000자의 문자열입니다.

#### Response

셀러 등록에 성공하면 JWE로 암호화된 [Seller 객체](#seller-객체)가 돌아옵니다. 보안 키로 복호화해서 응답 값을 확인해주세요.

셀러 등록에 실패하면 JWE로 암호화된 [에러 객체](/reference/using-api/req-res#api-v2-에러-객체)가 돌아옵니다. 보안 키로 복호화해서 응답 값을 확인해주세요.

셀러 등록에서 [발생할 수 있는 에러](/reference/error-codes#셀러-등록)를 살펴보세요.

method: POST
path: /v2/sellers/\{id}

### 셀러 수정

등록된 셀러 정보를 수정합니다. 수정하고 싶은 필드만 Request Body 파라미터로 보내주세요. 파라미터를 보내지 않으면 정보가 바뀌지 않습니다. 데이터를 삭제하고 싶다면 파라미터의 값을 `null`로 보내주세요. 등록할 때 필수 값이었던 필드는 삭제할 수 없습니다.

셀러 수정은 지급대행 서비스의 POST 요청입니다. [`ENCRYPTION` 보안](/guides/v2/payouts#encryption-보안)으로 요청을 보내주세요.

#### Path 파라미터

- **id** 수정하고 싶은 Seller 객체의 `id`입니다. 토스페이먼츠에서 발급한 값이며, 오픈마켓 상점에서 발급한 `refSellerId`와 다른 값입니다.

#### Request Body 파라미터

- **company** 법인사업자 또는 개인사업자 셀러 정보입니다.
  - **name** 사업자명입니다.
  - **representativeName** 대표자명입니다.
  - **email** 사업자 이메일입니다.
  - **phone** 사업자 전화번호입니다. `-` 없이 숫자로만 구성된 최소 8자의 문자열입니다.

- **individual** 개인 셀러 정보입니다.
  - **email** 이메일입니다.
  - **phone** 전화번호입니다. `-` 없이 숫자로만 구성된 최소 8자의 문자열입니다.

- **account** 셀러에서 정산 금액을 지급받을 계좌 정보입니다.
  - **bankCode** 계좌의 은행 코드입니다. 은행 코드와 증권사 코드를 참고하세요.
  - **accountNumber** 계좌번호입니다. `-` 없이 숫자만 입력해주세요. 최대 길이는 14자입니다.
  - **holderName** 계좌 예금주입니다. 최대 길이는 공백을 포함해 한영 구분 없이 50자입니다.

- **metadata** 셀러와 관련된 추가 정보를 key-value 쌍으로 담고 있는 JSON 객체입니다. 최대 5개의 키-값(key-value) 쌍입니다. 키는 `[` , `]` 를 사용하지 않는 최대 40자의 문자열, 값은 최대 2000자의 문자열입니다.

#### Response

셀러 수정에 성공하면 JWE로 암호화된 [Seller 객체](#seller-객체)가 돌아옵니다. 보안 키로 복호화해서 응답 값을 확인해주세요.

셀러 수정에 실패하면 JWE로 암호화된 [에러 객체](/reference/using-api/req-res#api-v2-에러-객체)가 돌아옵니다. 보안 키로 복호화해서 응답 값을 확인해주세요.

셀러 수정에서 [발생할 수 있는 에러](/reference/error-codes#셀러-수정)를 살펴보세요.

method: DELETE
path: /v2/sellers/\{id}

### 셀러 삭제

등록된 셀러를 삭제합니다. 삭제한 셀러의 `id` 식별자는 다시 발급되지 않고 조회가 불가능합니다.

#### Path 파라미터

- **id** 삭제하고 싶은 Seller 객체의 `id`입니다. 토스페이먼츠에서 발급한 값이며, 오픈마켓 상점에서 발급한 `refSellerId`와 다른 값입니다.

#### Response

셀러 삭제에 성공하면 [`deleted-entity` 객체](/reference/using-api/req-res#삭제된-객체)가 돌아옵니다.

셀러 삭제에 실패하면 [에러 객체](/reference/using-api/req-res#api-v2-에러-객체)가 돌아옵니다.

셀러 삭제에서 [발생할 수 있는 에러](/reference/error-codes#셀러-삭제)를 살펴보세요.

method: GET
path: /v2/sellers/\{id}

### 셀러 단건 조회

토스페이먼츠에 등록한 하나의 셀러를 조회합니다. 삭제되지 않은 셀러만 조회할 수 있어요.

#### Path 파라미터

- **id** 조회하고 싶은 Seller 객체의 `id`입니다. 토스페이먼츠에서 발급한 값이며, 오픈마켓 상점에서 발급한 `refSellerId`와 다른 값입니다.

#### Response

셀러 단건 조회에 성공하면 [Seller 객체](#seller-객체)가 돌아옵니다.

셀러 단건 조회에 실패하면 [에러 객체](/reference/using-api/req-res#api-v2-에러-객체)가 돌아옵니다.

셀러 단건 조회에서 [발생할 수 있는 에러](/reference/error-codes#셀러-단건-조회)를 살펴보세요.

method: GET
path: /v2/sellers

### 셀러 목록 조회

토스페이먼츠에 등록한 셀러 목록을 조회합니다. 삭제되지 않은 셀러만 조회할 수 있어요.

\* 라이브 환경에서 비정상적으로 많은 요청을 보내면 `429 Too Many Requests` 상태 코드가 내려올 수 있습니다.

#### Query 파라미터

- **limit** 조회하고 싶은 셀러 개수입니다. 기본 값은 `10`입니다. 설정할 수 있는 최대값은 `10000`입니다.
- **startingAfter** 특정 셀러의 `id`입니다. 해당 `id`의 셀러 다음으로 등록된 셀러부터 조회됩니다. 기본 값은 최초에 등록된 셀러입니다.

#### Response

셀러 목록 조회에 성공하면 [`seller-list`](/reference/using-api/req-res#리스트-객체) 객체가 돌아옵니다. `entityBody`에 다음과 같은 필드가 있습니다.

name: hasMore
type: boolean

더 조회할 수 있는 데이터가 있는지 알려줍니다. 마지막 데이터라면 `false`, 다음 데이터가 있다면 `true`입니다.

name: size
type: integer

조회된 객체의 개수입니다. `items`의 크기입니다.

name: nextCursor
type: stirng

다음 조회 시 `startingAfter`로 설정하면 되는 커서입니다. 만약에 더 조회할 데이터 없다면 `null`입니다.

name: items
type: array

[Seller 객체](#seller-객체) 목록입니다.

셀러 목록 조회에 실패하면 [에러 객체](/reference/using-api/req-res#api-v2-에러-객체)가 돌아옵니다.

셀러 목록 조회에서 [발생할 수 있는 에러](/reference/error-codes#셀러-단건-조회)를 살펴보세요.

### Payout 객체

지급대행 요청 정보입니다.

#### 객체 상세

- **id** 토스페이먼츠에서 발급하는 지급대행 요청의 고유 식별자입니다. 지급대행 취소, 조회 등에 사용됩니다. 최대 길이는 35자입니다.

- **refPayoutId** `nullable` 지급대행 상점에서 직접 발급하는 지급대행 요청의 고유 식별자입니다. 연동 중에 참고할 수 있는 값입니다. 등록 이후에 수정할 수 없고, 같은 값을 다시 사용할 수 없습니다. 최대 길이는 50자입니다.

- **destination** 정산금을 지급받는 셀러입니다. [Seller 객체](#seller-객체)의 `id` 값입니다.

- **scheduleType** 지급대행의 유형입니다. 아래 둘 중 하나입니다.

  \- `EXPRESS`: 바로지급입니다. 요청한 당일 이내 정산이 지급됩니다.

  \- `SCHEDULED`: 예약지급입니다. 요청일을 설정해서 정산 지급을 예약합니다.

- **payoutDate** `yyyy-MM-dd` 형식의 지급일입니다.

- **amount** 셀러에게 지급하는 정산 금액입니다.
  - **currency** 통화입니다. 현재는 `KRW`만 지원합니다.
  - **value** 잔액입니다.

- **transactionDescription** 이체 내역에 표시되는 적요입니다. 최대 길이는 7자입니다.

- **requestedAt** 지급대행이 요청된 시점입니다. `yyyy-MM-dd'T'HH:mm:ss±hh:mm` 형식입니다.

- **status** 지급대행 상태입니다. 지급대행 상태가 바뀌면 [`payout.changed`](/reference/using-api/webhook-events#payoutchanged) 웹훅이 발송됩니다.

  \- `REQUESTED`: 지급이 요청되었지만 아직 처리되지 않은 상태입니다. `REQUESTED` 상태일 때만 지급대행 요청을 취소할 수 있습니다.

  \- `IN_PROGRESS`: 지급을 처리하고 있는 상태입니다.

  \- `COMPLETED`: 셀러에 지급이 완료된 상태입니다.

  \- `FAILED`: 지급 요청이 실패한 상태입니다.

  \- `CANCELED`: 지급 요청이 취소된 상태입니다.

  \- `DELETED`: 지급요청서가 삭제된 상태입니다. 웹훅이 발송되지 않습니다.

  \- `REJECTED`: 지급 요청이 반려된 상태입니다. 웹훅은 `FAILED` 상태로 발송됩니다.

- **error** `nullable` 지급대행 요청에서 일어난 에러 객체입니다.

- **metadata** `nullable` 지급과 관련된 추가 정보를 key-value 쌍으로 담고 있는 객체입니다. 최대 5개의 키-값(key-value) 쌍입니다. 키는 `[` , `]` 를 사용하지 않는 최대 40자의 문자열, 값은 최대 2000자의 문자열입니다.

method: POST
path: /v2/payouts

### 지급대행 요청

지급 대상 셀러에게 정산 금액을 이체합니다. 요청 시점에 잔액이 부족하면 지급이 실패합니다.

지급대행 요청은 지급대행 서비스의 POST 요청입니다. [`ENCRYPTION` 보안](/guides/v2/payouts#encryption-보안)으로 요청을 보내주세요.

최대 100건의 지급대행을 한 번에 요청할 수 있습니다. 하지만 하나의 요청이 실패하면 요청 모두 실패하고 첫 번째 실패 건에 대한 에러 객체가 응답으로 돌아옵니다. 1건의 지급대행 요청에는 10억 미만의 정산 금액을 이체할 수 있습니다.

지급대행으로 셀러에게 지급한 정산 금액은 회수하기 어렵습니다. 토스페이먼츠는 지급 관련 책임이 없기 때문에 지급대행 서비스를 신중하게 사용해주세요.

\* 지급대행 요청 API를 라이브 환경에서 테스트하면 수수료가 부과됩니다.

#### Request Body 파라미터

- **refPayoutId** 상점에서 직접 발급하는 지급대행 식별자입니다. 각 지급대행 요청 건에 고유 값을 발급해야 되고 지급대행 상점에서 참고용으로 사용할 수 있습니다.

- **destination** 정산금을 지급받는 셀러입니다. [Seller 객체](#seller-객체)의 `id` 값입니다.

- **scheduleType** 지급대행의 유형입니다. 아래 둘 중 하나입니다.

  \- `EXPRESS`: 바로지급입니다. 요청한 당일 이내 정산금이 지급됩니다. 바로지급를 요청할 수 있는 시간은 영업일 08:00~15:00입니다.

  \- `SCHEDULED`: 예약지급입니다. 요청일을 설정해서 정산금 지급을 예약합니다.

- **payoutDate** `yyyy-MM-dd` 형식의 지급일입니다. 지급대행 유형이 `SCHEDULED`일 때는 필수입니다. 지급일은 미래에 1년 이내에 있는 영업일로만 설정할 수 있어요. 휴일, 공휴일, 과거일자로 설정하면 오류가 발생해요.

- **amount** 지급 금액 정보입니다.

- **transactionDescription** 이체 내역에 표시되는 적요입니다. 최대 길이는 7자입니다.

- **metadata** 요청과 관련된 추가 정보를 key-value 쌍으로 담고 있는 객체입니다. 최대 5개의 키-값(key-value) 쌍입니다. 키는 `[` , `]` 를 사용하지 않는 최대 40자의 문자열, 값은 최대 2000자의 문자열입니다.

#### Response

지급대행 요청에 성공하면 JWE로 암호화된 [`payout-list` 객체](/reference/using-api/req-res#리스트-객체)가 돌아옵니다. 보안 키로 복호화해서 응답 값을 확인해주세요.

객체의 `entityBody` 필드의 items에서 [Payout 객체](#payout-객체) 목록을 확인할 수 있습니다.

지급대행 요청에 실패하면 JWE로 암호화된 [에러 객체](/reference/using-api/req-res#api-v2-에러-객체)가 돌아옵니다. 보안 키로 복호화해서 응답 값을 확인해주세요.

지급대행 요청에 실패하면 요청 건 모두 실패하고 요청 건 중에 첫 번째 에러에 대한 에러 객체가 돌아옵니다.

지급대행 요청에서 [발생할 수 있는 에러](/reference/error-codes#지급대행-요청)를 살펴보세요.

method: POST
path: /v2/payouts/\{id}/cancel

### 지급 요청 목록 조회

지급 요청 목록을 조회합니다.

\* 라이브 환경에서 비정상적으로 많은 요청을 보내면 `429 Too Many Requests` 상태 코드가 내려올 수 있습니다.

#### Query 파라미터

- **limit** 조회하고 싶은 지급대행 요청건의 개수입니다. 기본 값은 `10`이고, 설정할 수 있는 최대값은 `10000`입니다.

- **payoutDateGte** 커서로 사용할 지급일입니다. 해당 지급일과 같거나 더 이후(이상)에 발생하는 지급대행 요청 건을 조회합니다.
- **payoutDateLte** 지급일입니다. 해당 지급일과 같거나 더 이전(이하)에 발생하는 지급대행 요청 건을 조회합니다.

#### Response

지급대행 요청에 [`payout-list` 객체](/reference/using-api/req-res#리스트-객체)가 돌아옵니다. `entityBody`에 다음과 같은 필드가 있습니다.

name: hasMore
type: boolean

더 조회할 수 있는 데이터가 있는지 알려줍니다. 마지막 데이터라면 `false`, 다음 데이터가 있다면 `true`입니다.

name: size
type: integer

조회된 객체의 개수입니다. `items`의 크기입니다.

name: nextCursor
type: stirng

다음 조회 시 `startingAfter`로 설정하면 되는 커서입니다. 만약에 더 조회할 데이터 없다면 `null`입니다.

name: items
type: array

[Payout 객체](#payout-객체) 목록입니다.

지급 요청 목록 조회에 실패하면 [에러 객체](/reference/using-api/req-res#api-v2-에러-객체)가 돌아옵니다.
