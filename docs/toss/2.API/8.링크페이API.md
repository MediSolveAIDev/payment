## 상품

상품 API로 결제 링크에 사용할 상품을 생성하고 관리할 수 있습니다. [Basic 인증 방식](/reference/using-api/authorization#인증-헤더)을 사용합니다.

### Product 객체

상품 정보를 담고 있는 객체입니다.

#### 객체 상세

- **productKey** 상품의 고유 ID입니다.

- **name** 상품명입니다.

- **amount** 상품 가격입니다.

- **currency** 통화 정보입니다.

- **description** `nullable` 상품 설명입니다.

- **exchangeRefundPolicy** `nullable` 환불 정책입니다.

- **status** 상품 상태입니다. `ON_SALE`, `SOLD_OUT` 중 하나입니다.

- **url** `nullable` 결제 링크 URL입니다. 이 URL을 구매자에게 공유하면 결제를 진행할 수 있습니다.

- **useMemo** 구매자의 요청사항 입력란 사용 여부입니다.

- **validThrough** `nullable` 상품 유효기간입니다.

- **mainImage** `nullable` 메인 이미지 정보입니다.
  - **productImageKey** 이미지의 고유 ID입니다.

  - **path** 이미지 URL입니다.

- **images** 상세 이미지 리스트입니다.
  - **productImageKey** 이미지의 고유 ID입니다.

  - **path** 이미지 URL입니다.

- **optionDetail** 상품의 옵션 구성 정보입니다.
  - **groups** 옵션의 분류 기준 리스트입니다. 예를 들어 `사이즈`, `색상` 등입니다.
    - **id** 옵션 그룹 ID입니다.

    - **name** 옵션 그룹 이름입니다. 예를 들어 `사이즈`, `색상` 등입니다.

    - **values** 옵션 그룹 값 리스트입니다. 예를 들어 `["S", "M", "L"]` 등입니다.

  - **options** `groups`에서 정의한 값을 조합한 항목 리스트입니다. 예를 들어 `사이즈`와 `색상`을 조합한 `["M", "블랙"]`이 하나의 항목입니다. 각 항목별로 가격, 재고, 판매 가능 여부를 담고 있습니다.
    - **id** 항목의 고유 ID입니다.

    - **names** 조합된 옵션 그룹 값 리스트입니다. 예를 들어 `["M", "블랙"]`입니다.

    - **amount** 항목의 가격입니다.

    - **currency** 통화 정보입니다.

    - **available** 항목의 이용 가능 여부입니다.

    - **stock** `nullable` 항목의 재고 정보입니다.
      - **totalQuantity** 전체 재고 수량입니다.

      - **remainQuantity** 남은 재고 수량입니다.

- **stock** `nullable` 상품 재고 정보입니다.
  - **totalQuantity** 전체 재고 수량입니다.

  - **remainQuantity** 남은 재고 수량입니다.

- **deliveryPolicy** `nullable` 상품 배송 정보입니다.
  - **amount** 배송비입니다.

  - **currency** 통화 정보입니다.

  - **period** 배송 기간입니다. 예를 들어 `3일 이내`입니다.

- **createdAt** 상품 생성 일시입니다. `yyyy-MM-dd'T'HH:mm:ss±hh:mm` ISO 8601 형식입니다.

method: POST
path: /v1/products

### 상품(링크) 생성

결제 링크에 사용할 상품을 생성합니다.

#### Request Body 파라미터

- **name** 상품명입니다.

- **amount** 상품 가격입니다.

- **description** 상품 설명입니다.

- **stock** 상품 재고 정보입니다. 옵션 항목 재고와는 동시에 설정할 수 없습니다.
  - **quantity** 전체 재고 수량입니다.

- **optionDetail** 상품의 옵션 구성 정보입니다.
  - **groups** 옵션의 분류 기준 리스트입니다. 예를 들어 `사이즈`, `색상` 등입니다.
    - **name** 옵션 그룹 이름입니다.
    - **values** 옵션 그룹 값 리스트입니다.
  - **options** `groups`에서 정의한 값을 조합한 항목 리스트입니다. 예를 들어 `사이즈`와 `색상`을 조합한 `["M", "블랙"]`이 하나의 항목입니다.
    - **names** 조합된 옵션 그룹 값 리스트입니다. `groups`의 값에 포함된 값이어야 합니다.
    - **amount** 항목의 가격입니다.
    - **available** 항목의 이용 가능 여부입니다.
    - **stock** 항목의 재고 정보입니다.
      - **quantity** 전체 재고 수량입니다.

- **useMemo** 구매자의 요청사항 입력란 사용 여부입니다.

- **validThrough** 상품 유효기간입니다.

- **exchangeRefundPolicy** 환불 정책입니다.

- **deliveryPolicy** 상품 배송 정보입니다.
  - **amount** 배송비입니다.
  - **period** 배송 기간입니다.

#### Response

상품 생성에 성공했다면 [Product 객체](#product-객체)가 돌아옵니다.

상품 생성에 실패했다면 HTTP 상태 코드와 함께 [에러 객체](/reference/using-api/req-res#에러-객체)가 돌아옵니다.

[상품(링크) 생성 API에서 발생할 수 있는 에러](/reference/error-codes#상품링크-생성)를 살펴보세요.

method: POST
path: /v1/products/\{productKey}/sms

### 상품(링크) SMS 발송

상품의 결제 링크를 SMS로 발송합니다. 발신번호는 수정 불가합니다.

테스트 환경에서는 문자가 실제로 발송되지 않습니다.

#### Path 파라미터

- **productKey** 상품의 고유 ID입니다.

#### Request Body 파라미터

- **recipients** 문자를 수신할 핸드폰 번호 리스트입니다.
- **queryParams** 상품 결제 링크에 추가할 커스텀 쿼리 파라미터입니다. key-value 형태로 객체로 보내세요. 설정한 값은 주문 완료 후 [Order 객체](#order-객체)의 `metaFields`에 포함되어 돌아옵니다. JSON 변환 시 500자를 초과할 수 없습니다.

#### Response

문자 발송에 성공했다면 HTTP 200 상태 코드가 돌아옵니다.

문자 발송에 실패했다면 HTTP 상태 코드와 함께 [에러 객체](/reference/using-api/req-res#에러-객체)가 돌아옵니다.

[상품(링크) SMS 발송 API에서 발생할 수 있는 에러](/reference/error-codes#상품링크-sms-발송)를 살펴보세요.

method: GET
path: /v1/products/\{productKey}

path: /v1/products

### 상품 목록 조회

상품 목록을 조회합니다.

#### Query 파라미터

- **limit** 한 번에 조회할 상품 수입니다. 최대 `100`까지 설정할 수 있으며 기본값은 `20`입니다.

#### Response

상품 목록 조회에 성공했다면 [Product 객체](#product-객체) 배열이 돌아옵니다.

상품 목록 조회에 실패했다면 HTTP 상태 코드와 함께 [에러 객체](/reference/using-api/req-res#에러-객체)가 돌아옵니다.

[상품 목록 조회 API에서 발생할 수 있는 에러](/reference/error-codes#상품-목록-조회)를 살펴보세요.

method: POST
path: /v1/products/\{productKey}

method: POST
path: /v1/products/\{productKey}/images

### 상품 이미지 업로드

상품에 이미지를 업로드합니다. `multipart/form-data` 형식으로 `files` 필드에 이미지 파일을 담아 요청합니다. 최대 10개까지 업로드할 수 있으며, 파일당 최대 10MB까지 가능합니다.

#### Path 파라미터

- **productKey** 상품의 고유 ID입니다.

#### Query 파라미터

- **imageType** 이미지 타입입니다. `MAIN`, `DETAIL` 중 하나입니다.

#### Response

이미지 업로드에 성공했다면 [ProductImage 객체](#productimage-객체) 배열이 돌아옵니다.

이미지 업로드에 실패했다면 HTTP 상태 코드와 함께 [에러 객체](/reference/using-api/req-res#에러-객체)가 돌아옵니다.

[이미지 업로드 API에서 발생할 수 있는 에러](/reference/error-codes#이미지-업로드)를 살펴보세요.

## 주문

주문 API로 결제 링크를 통해 발생한 주문을 조회하고 관리할 수 있습니다. [Basic 인증 방식](/reference/using-api/authorization#인증-헤더)을 사용합니다.

### Order 객체

주문 정보를 담고 있는 객체입니다.

#### 객체 상세

- **orderKey** 주문의 고유 ID입니다.

- **amount** 주문 금액입니다.

- **currency** 통화 정보입니다.

- **metaFields** 결제 링크의 `queryParams`로 전달된 데이터가 담긴 필드입니다.

- **orderItems** 주문 아이템 리스트입니다.
  - **product** 상품 정보입니다. [Product 객체](#product-객체)를 참고하세요.

  - **quantity** 상품 수량입니다.

  - **orderOptions** 주문 옵션 리스트입니다.
    - **names** 옵션 이름 리스트입니다.

    - **amount** 옵션 가격입니다.

    - **currency** 통화 정보입니다.

- **customerName** `nullable` 구매자 이름입니다.

- **customerPhoneNumber** `nullable` 구매자 핸드폰 번호입니다.

- **memo** `nullable` 구매자의 요청사항입니다.

- **shipping** `nullable` 구매자 배송 정보입니다.
  - **receiver** 수취인입니다.

  - **postalCode** 우편번호입니다.

  - **address** 기본주소입니다.

  - **detailAddress** 상세주소입니다.

  - **fullAddress** 전체 주소입니다.

- **payment** `nullable` 결제 정보입니다. 결제 승인이 완료되면 값이 들어옵니다. [Payment 객체](/reference#payment-객체)를 참고하세요.

- **createdAt** 주문 생성 일시입니다. `yyyy-MM-dd'T'HH:mm:ss±hh:mm` ISO 8601 형식입니다.

method: GET
path: /v1/orders

### 주문 목록 조회

주문 목록을 조회합니다.

#### Query 파라미터

- **limit** 한 번에 조회할 주문 수입니다. 최대 `100`까지 설정할 수 있으며 기본값은 `20`입니다.

#### Response

주문 목록 조회에 성공했다면 [Order 객체](#order-객체) 배열이 돌아옵니다.

주문 목록 조회에 실패했다면 HTTP 상태 코드와 함께 [에러 객체](/reference/using-api/req-res#에러-객체)가 돌아옵니다.

[주문 목록 조회 API에서 발생할 수 있는 에러](/reference/error-codes#주문-목록-조회)를 살펴보세요.

method: GET
path: /v1/orders/\{orderKey}

### 주문 취소

주문을 취소합니다.

#### Path 파라미터

- **orderKey** 취소할 주문의 고유 ID입니다.

#### Request Body 파라미터

- **cancelReason** 취소 사유입니다.

- **cancelAmount** 취소 금액입니다. 부분 취소 시 사용합니다.

- **currency** 통화 정보입니다.

- **taxFreeAmount** 면세 금액입니다.

- **refundReceiveAccount** 가상계좌 결제 취소 시 환불 계좌 정보입니다.
  - **bank** 은행 한글명입니다.
  - **accountNumber** 계좌번호입니다.
  - **holderName** 예금주명입니다.

#### Response

주문 취소에 성공했다면 [Order 객체](#order-객체)가 돌아옵니다.

주문 취소에 실패했다면 HTTP 상태 코드와 함께 [에러 객체](/reference/using-api/req-res#에러-객체)가 돌아옵니다.

[주문 취소 API에서 발생할 수 있는 에러](/reference/error-codes#주문-취소)를 살펴보세요.
