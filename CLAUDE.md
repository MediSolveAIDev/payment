구독,결제 서버 API

- fastapi, postgresql, redis 사용
- API서버 admin기능 화면을 위한 htmx 사용
- 결제 모듈: tosspayment
  - 참고문서: docs/toss
- 개요
  - 사내 다양한 서비스에서 구독 및 결제를 사용하려고 함
  - 사내 서비스에서 사용할 구독/결제 API서버 구축
  - htmx에서는 로그인해서 구독정보를 관리
    - 관리자관리
    - 서비스관리 - 서비스관리 내에서 요금제 관리
    - 구독정보 - 실제 구독한 내역을 확인
- 프로세스
  - 구독/결제서비스를 이용하려는 서비스정보를 등록
    - 서비스명
    - 서비스 ip
    - 구독/결제 관리할 담당자 email
  - 등록 후 서비스 key를 생성 -> 서비스 담당자에게 전달
    - 서비스 key를 서비스 정보에 저장
  - 서비스 담당자가 로그인
  - 구독요금제 생성
  - 외부 서비스에서 구독요청 -> 구독처리
  - 구독은 만료일이 되면 자동연장
- 규칙
  - 서비스, 사용자 당 1개 구독만 가능
  - 구독취소 -> 만료일되면 자동 종료
  - 요금제, 서비스 삭제는 구독이 있는 경우 삭제가 안됨
  - 결제주기: 년,월,주,일 (일을 선택하면 원하는 일수를 선택)
  - 첫구독은 무료 및 할인금액 적용 기능
  - 외부 사용자 식별자(external_user_id)는 **이메일만** 사용 — 서버가 소문자/공백 정규화해 저장·조회(app/core/identifiers.py)
